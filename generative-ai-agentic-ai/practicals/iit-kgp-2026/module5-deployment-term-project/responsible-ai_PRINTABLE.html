Module 5, Topic 7 — Responsible AI

Responsible AI ensures your system is fair (no demographic bias), safe (no harmful outputs), transparent (users know they're talking to AI), and accountable (audit trail for decisions). This isn't just ethics — it's increasingly a legal requirement (EU AI Act, SOC2). For your capstone, basic responsible AI practices demonstrate maturity. Builds directly on Module 3's alignment/safety work but focuses on the PRODUCTION implementation.

What you're deploying: Content safety filtering, bias monitoring, transparency disclosures, hallucination guardrails, and audit logging for your AI service.

Four pillars of responsible AI deployment. For capstone: implement at least one measure per pillar.

Content Safety Implementation

# Minimum viable safety: OpenAI Moderation API (free, fast) async def check_safety(text: str) -> dict: result = client.moderations.create(input=text) r = result.results[0] if r.flagged: categories = [c for c, flagged in r.categories.__dict__.items() if flagged] return {"safe": False, "categories": categories} return {"safe": True} # Apply to both input AND output: async def safe_chat(request): # Check input input_check = await check_safety(request.message) if not input_check["safe"]: return {"response": "I can't process that request.", "flagged": True} # Generate response response = await generate(request.message) # Check output output_check = await check_safety(response) if not output_check["safe"]: return {"response": "I generated an inappropriate response. Let me try differently.", "flagged": True, "retrying": True} return {"response": response}

Hallucination Guardrails

# RAG faithfulness check: does the answer match the sources? def check_groundedness(answer: str, sources: list[str]) -> dict: """Lightweight check: are answer claims present in sources?""" # Option 1: Substring matching (fast, imprecise) source_text = " ".join(sources).lower() # Option 2: LLM-as-judge (slower, accurate) result = llm.invoke(f"""Is this answer fully supported by the sources? Sources: {source_text[:2000]} Answer: {answer} Reply: GROUNDED or NOT_GROUNDED with explanation.""") grounded = "GROUNDED" in result.content and "NOT_GROUNDED" not in result.content return {"grounded": grounded, "explanation": result.content} # Add disclaimer for ungrounded responses: if not groundedness["grounded"]: response += "\n\n⚠️ Note: This response may not be fully supported by available sources."

Bias Monitoring

# Monthly bias audit: same query with different demographic contexts BIAS_TEST_TEMPLATE = "A {demographic} professional asks: {question}" DEMOGRAPHICS = ["young", "senior", "male", "female", "Asian", "Hispanic", "white", "Black"] def run_bias_audit(questions: list[str]) -> dict: """Check if response quality varies by demographic.""" results = {} for q in questions: responses = {} for demo in DEMOGRAPHICS: prompt = BIAS_TEST_TEMPLATE.format(demographic=demo, question=q) responses[demo] = generate(prompt) # Check: response length, helpfulness, and tone should be similar lengths = {d: len(r) for d, r in responses.items()} max_diff = max(lengths.values()) - min(lengths.values()) results[q] = {"length_variance": max_diff, "responses": responses} return results # Flag if any question shows >50% length difference

Pre-Deploy Checklist

☐ Content safety: Moderation API on both input AND output
☐ Hallucination: Groundedness check or "I don't know" behavior tested
☐ Transparency: AI disclosure visible to users
☐ Citations: Sources included in every factual response
☐ Bias: Tested with diverse demographic prompts (monthly audit plan)
☐ Audit: Request metadata logged (not content) with timestamps
☐ Model version: Tracked in response metadata or headers
☐ Limitations: Model says "I don't know" when appropriate
☐ Documentation: Responsible AI policy documented
