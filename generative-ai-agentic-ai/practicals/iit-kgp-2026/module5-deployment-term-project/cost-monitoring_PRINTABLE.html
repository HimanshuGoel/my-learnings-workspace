Module 5, Topic 6 — Cost Monitoring

LLM API costs scale linearly with usage and can spike unexpectedly — an agent that loops, a user who sends huge prompts, or a caching failure can turn a $5/day service into a $500/day bill overnight. Unlike traditional APIs (fixed infra cost), AI services have VARIABLE per-request costs determined by token count. You need real-time tracking, per-user budgets, and alerts BEFORE you hit the bill.

What you're deploying: Token usage tracking per request/user/endpoint, daily budget alerts, cost attribution dashboards, and automatic throttling when limits are reached.

Cost Structure

AI costs are VARIABLE per request. Track tokens, not just request count. Output tokens cost 4× more than input.

Implementation — Token Tracking

from dataclasses import dataclass, field from collections import defaultdict from datetime import date import tiktoken @dataclass class UsageRecord: input_tokens: int = 0 output_tokens: int = 0 cost_usd: float = 0.0 requests: int = 0 class CostTracker: PRICING = { "gpt-4o-mini": {"input": 0.15e-6, "output": 0.60e-6}, "gpt-4o": {"input": 2.50e-6, "output": 10.00e-6}, "text-embedding-3-small": {"input": 0.02e-6, "output": 0}, } def __init__(self, daily_budget: float = 50.0): self.daily_budget = daily_budget self.usage: dict[str, dict[str, UsageRecord]] = defaultdict(lambda: defaultdict(UsageRecord)) def record(self, user_id: str, model: str, input_tokens: int, output_tokens: int): pricing = self.PRICING.get(model, {"input": 0, "output": 0}) cost = input_tokens * pricing["input"] + output_tokens * pricing["output"] today = str(date.today()) rec = self.usage[today][user_id] rec.input_tokens += input_tokens rec.output_tokens += output_tokens rec.cost_usd += cost rec.requests += 1 # Check budget total_today = sum(r.cost_usd for r in self.usage[today].values()) if total_today > self.daily_budget: raise BudgetExceeded(f"Daily budget ${self.daily_budget} exceeded: ${total_today:.2f}") return cost
