Topic 4 — Attention Mechanism

Summary: Attention lets every token look at every other token simultaneously, computing relevance scores to build context-aware representations. It's the core innovation that made Transformers possible.

Why It Exists

  • RNN problem — processes one token at a time sequentially. Long-range info gets lost. Can't parallelize. Slow.
  • Attention solution — every token directly looks at every other token. No distance limitation. Fully parallel (GPU-friendly).
  • Mental Model — a smart search engine inside the model. For each token: "Which other tokens are relevant to me?" → get weighted combination of relevant ones.
  • Software analogy — like event listeners. Each token "listens" to all others, but only "responds" strongly to relevant ones. Or: a soft database JOIN (similarity-based, not exact key match).

The Core Idea: Q, K, V

Every token asks a Query, offers a Key, and provides a Value. Scores = Q·K similarity. Output = weighted sum of Values.

Step-by-Step Visual

Attention computes relevance of each token to the current one, then combines their values proportionally.

Multi-Head Attention

  • What — run 12 (or more) parallel attention computations, each with its own Q/K/V weights.
  • Why — different heads learn different relationship types: one for syntax, one for semantics, one for proximity, etc.
  • How — concatenate all head outputs → linear projection → final output.
Multi-Head = [head_1 | head_2 | ... | head_12] × W_output Each head: Attention(Q×W_Qi, K×W_Ki, V×W_Vi)

Why O(n²) Matters

Sequence LengthAttention PairsImpact
512 tokens262K pairsFast (standard)
4K tokens16M pairsManageable
128K tokens16B pairsExpensive! (why long-context models cost more)
This is why context windows have limits. Doubling sequence length = 4× compute. Long-context models (128K) use optimizations like FlashAttention.

Before vs After Attention

AspectRNN/LSTM (before)Attention (after)
ProcessingSequential (one token at a time)Parallel (all tokens at once)
Long-range depsForgotten after ~20 tokensDirect access to any token
SpeedSlow (can't parallelize)Fast on GPU (matrix multiply)
BottleneckOne fixed-size hidden stateNo bottleneck — attends to everything

Common Misconceptions

WrongCorrect
Attention = understandingIt's a mathematical weighted average. No guaranteed "understanding."
Each head has a specific roleHeads learn patterns empirically — not designed for specific purposes.
Attention is O(n)It's O(n²) in sequence length — why long contexts are expensive.
Attention weights = explanationWeights are intermediate computations. Don't over-interpret them.

Key Terminology

TermMeaning
Self-attentionToken attends to other tokens in the SAME sequence
Cross-attentionToken attends to tokens in a DIFFERENT sequence (e.g., encoder→decoder)
Multi-headMultiple parallel attention computations (different perspectives)
Causal/Masked attentionCan only attend to PREVIOUS tokens (used in GPT — can't peek ahead)
Attention scoreHow relevant one token is to another (Q·K dot product)
