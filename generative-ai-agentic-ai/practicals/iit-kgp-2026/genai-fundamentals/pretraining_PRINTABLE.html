Topic 6 — Pre-training

Summary: Pre-training is the expensive one-time process of teaching a Transformer language by predicting next tokens on internet-scale data — producing a general-purpose model that you later fine-tune cheaply.

Why It Exists

  • Problem — training a model from scratch per task needs enormous labeled datasets. Most tasks don't have enough labeled data.
  • Solution — train ONCE on unlabeled internet text (essentially infinite), then adapt cheaply to any task. General knowledge → specific skills.
  • Mental Model — University education. A degree (pre-training) gives broad knowledge. Then you specialize for a job (fine-tuning). You don't re-learn reading for every project.
  • Or: Pre-compiling a shared library. Expensive compilation done once; every app uses it without recompiling.

How It Works

Pre-training: predict next token billions of times on internet text → model learns language

Training Objectives

ObjectiveModelsHow
Causal LM (next-token)GPT, LLaMA, Claude"The cat sat on ___" → predict "mat"
Masked LM (fill blanks)BERT"The [MASK] sat on mat" → predict "cat"
Seq2Seq (denoise)T5Corrupt text → reconstruct
Causal LM dominates today. All modern generative models (GPT-4, Claude, LLaMA) use next-token prediction. BERT-style is for understanding tasks (classification, NER).

What the Model Learns (implicitly)

  • Grammar and syntax (what's valid English/code)
  • World knowledge (facts common in training data)
  • Reasoning patterns (how conclusions follow premises)
  • Code patterns (functions, APIs, debugging approaches)
  • Multi-language capability
  • Style/format awareness (emails vs code vs essays)

The Full Recipe: Pre-training → Instruction Tuning → RLHF

Raw pre-trained model just completes text. Instruction tuning + RLHF makes it a useful assistant.

You don't pre-train. Labs do (OpenAI, Meta, Google). You use or fine-tune the result. Your Module 3 work starts from a pre-trained checkpoint.

Common Misconceptions

WrongCorrect
"Pre-training = fine-tuning"Pre-training = general (months, $M). Fine-tuning = specific task (hours, $10s).
"Model memorizes text"Learns patterns/distributions. Some memorization, but mostly generalization.
"More data always better"Data QUALITY matters enormously. Junk in = junk out.
"Pre-trained = ready to use"Raw model just completes text. Needs instruction tuning to follow prompts.
"I should pre-train my own"Almost never. Use existing + fine-tune. Pre-training = lab-scale budgets.

Key Terminology

TermMeaning
Pre-trainingInitial training on internet text (expensive, general)
Self-supervisedLabels come from the data itself (next token IS the label)
Knowledge cutoffDate training data ends — model doesn't know newer events
Base modelRaw pre-trained model before instruction tuning
Instruction tuningTraining on Q&A pairs so model follows instructions
RLHFReinforcement Learning from Human Feedback — makes model helpful/safe
Transfer learningReusing knowledge from pre-training for new tasks
