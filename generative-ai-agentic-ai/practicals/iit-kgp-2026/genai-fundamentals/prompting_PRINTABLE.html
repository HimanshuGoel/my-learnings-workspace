Topic 7 — Prompting

Summary: Prompting is configuring an LLM's behavior through natural language input — system instructions, examples, and reasoning hints — without changing the model's weights. It's the cheapest way to steer an expensive model.

Why It Exists

  • Problem — you have a powerful pre-trained model. How to make it do YOUR specific task?
  • Two options: Fine-tune (expensive, slow, needs data) OR Prompt (free, instant, iterate in seconds).
  • Mental Model — Writing a clear ticket for a smart but literal contractor. Vague ticket → vague output. Specific, structured ticket → precise output. Or: configuring Spring Boot via application.yml — you don't change the framework, you configure it.

Prompting Strategies (Ladder of Power)

Prompting strategies increase in power and complexity from left to right

Zero-Shot (just instructions)

Classify the sentiment as positive, negative, or neutral: "The food was amazing but the service was terrible"

Few-Shot (show examples)

Classify sentiment: "Loved it!" → positive "Terrible" → negative "It was okay" → neutral "The food was amazing but service was terrible" →

Chain-of-Thought (think step by step)

Solve step by step: If a shirt costs $20 and is 25% off, what's the final price? Step 1: Discount = $20 × 0.25 = $5 Step 2: Final = $20 - $5 = $15 Answer: $15
Magic phrase: Adding "Let's think step by step" dramatically improves reasoning on math, logic, and multi-step problems.

System Prompt (set persona + constraints)

System: You are a senior Python developer. Write clean, documented code with error handling and type hints. User: Write a function to validate an email address.

Structured Output (force format)

Extract from this text. Return as JSON: {"name": "...", "email": "...", "sentiment": "...", "confidence": 0.0-1.0} Text: "Hi, I'm Alice (alice@example.com). I love your product!"

Common Misconceptions

WrongCorrect
"One correct prompt exists"Prompting is iterative — multiple phrasings work, test and compare
"Longer = better"Concise clarity often outperforms verbose prompts
"CoT always helps"Overkill for simple tasks (wastes tokens). Use for reasoning only.
"Prompts are permanent"Model updates can break them. Version and test continuously.
"Prompting replaces fine-tuning"For high-volume specific tasks, fine-tuning is cheaper per call

Top Takeaways

3Few-shot examples are the easiest way to improve output quality
4"Think step by step" unlocks reasoning on complex problems (CoT)
