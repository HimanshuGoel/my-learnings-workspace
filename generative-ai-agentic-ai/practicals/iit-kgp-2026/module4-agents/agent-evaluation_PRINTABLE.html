Module 4, Topic 9 — Agent Evaluation

Agents are non-deterministic — can't test like functions. Must evaluate TRAJECTORIES (sequence of decisions and actions) not just final output. Key metrics: task completion, tool accuracy, step efficiency, reasoning quality. Run tests multiple times to detect flaky behavior. LLM-as-judge handles subjective quality that rules can't capture.

Problem Statement: A function test says "given input X, output should be Y." Agent tests are harder: the agent might take different paths each run but still reach the right answer. We need to evaluate not just WHAT it produces but HOW it gets there — detecting loops, wrong tool choices, and inefficiency alongside incorrect answers.

Evaluation Architecture

Agent evaluation pipeline: Test Case → Agent → Multiple Evaluators → Aggregated Metrics

Trajectory vs Output-Only Evaluation

Trajectory eval catches inefficiency and wrong paths even when the final answer happens to be correct

Implementation Code

Trajectory Capture

@dataclass class AgentTrajectory: query: str steps: list[TrajectoryStep] = field(default_factory=list) final_output: str = "" tools_called: list[str] = field(default_factory=list) total_time_ms: float = 0 @property def step_count(self) -> int: return len(self.steps) class TrajectoryCapture: def __init__(self, agent): self.agent = agent def invoke_with_trace(self, query: str) -> AgentTrajectory: trajectory = AgentTrajectory(query=query) start = datetime.now() for event in self.agent.stream({"messages": [HumanMessage(query)]}): for node_name, output in event.items(): trajectory.steps.append(TrajectoryStep(node=node_name, ...)) # Track tool calls for msg in output.get("messages", []): if hasattr(msg, "tool_calls") and msg.tool_calls: trajectory.tools_called.append(msg.tool_calls[0]["name"]) trajectory.total_time_ms = (datetime.now() - start).total_seconds() * 1000 return trajectory

Evaluation Harness

@dataclass class TestCase: query: str expected_output_contains: list[str] expected_tools: list[str] = None forbidden_tools: list[str] = None max_steps: int = 10 def evaluate_trajectory(traj: AgentTrajectory, tc: TestCase) -> EvalScore: # Output correctness output_ok = all(exp.lower() in traj.final_output.lower() for exp in tc.expected_output_contains) # Tool accuracy tools_ok = (all(t in traj.tools_called for t in (tc.expected_tools or [])) and not any(t in traj.tools_called for t in (tc.forbidden_tools or []))) # Step budget budget_ok = traj.step_count <= tc.max_steps return EvalScore(output_ok, tools_ok, budget_ok)

Regression Suite (Run Multiple Times)

class AgentTestSuite: def run(self, test_cases: list[TestCase], runs_per_case: int = 5) -> dict: summary = {"total": 0, "passed": 0, "failed": 0, "flaky": 0} for tc in test_cases: pass_count = sum(1 for _ in range(runs_per_case) if evaluate_trajectory(capture.invoke_with_trace(tc.query), tc).all_pass) summary["total"] += 1 if pass_count == runs_per_case: summary["passed"] += 1 elif pass_count == 0: summary["failed"] += 1 else: summary["flaky"] += 1 # Non-determinism detected! return summary # Example: {"total": 50, "passed": 42, "failed": 3, "flaky": 5}

LLM-as-Judge

class TrajectoryJudgment(BaseModel): score: float = Field(ge=0.0, le=1.0) reasoning: str issues: list[str] = Field(default_factory=list) judge_llm = ChatOpenAI(model="gpt-4o").with_structured_output(TrajectoryJudgment) def llm_judge(trajectory: AgentTrajectory) -> TrajectoryJudgment: steps_text = "\n".join(f"Step {i+1}: {s.action}" for i, s in enumerate(trajectory.steps)) return judge_llm.invoke([ SystemMessage(content="Rate 0.0-1.0. Consider: logical path, no wasted steps, correct."), HumanMessage(content=f"Query: {trajectory.query}\nSteps:\n{steps_text}\nOutput: {trajectory.final_output}")])

Architecture Decision: Evaluate on trajectories (not just output) using deterministic checks (tools, budget) + LLM-as-judge (reasoning quality). Output-only misses efficiency problems. Tool-only misses creative solutions. LLM-as-judge handles subjective quality. The combination gives confidence across all dimensions. Trade-off: LLM-as-judge is expensive and non-deterministic — use sparingly (weekly), calibrate with human scores.
