Module 4, Topic 2 — Multimodal RAG

Real documents have tables, charts, diagrams, and mixed layouts that standard text-only RAG misses entirely. Multimodal RAG extends pipelines to handle PDFs by extracting text AND visual elements, describing images with VLMs for embedding, then attaching original images at generation time for maximum answer quality. The key insight: embed text descriptions for retrieval, but send actual images to the VLM for generation.

Problem Statement: Standard RAG only handles clean text. When your knowledge base contains PDFs with charts, tables, and diagrams, text extraction loses critical visual information. Multimodal RAG bridges this by combining text retrieval with visual understanding — ensuring the AI can reason over charts and tables just as a human reader would.

Three Approaches Comparison

Three approaches to multimodal RAG — Hybrid of Approach 1 + 3 gives best quality/cost balance

Recommended Architecture (Hybrid Pipeline)

Recommended hybrid architecture — VLM descriptions for indexing + original images attached at generation time

Implementation Code

Step 1: Document Parsing (Unstructured)

from unstructured.partition.pdf import partition_pdf elements = partition_pdf( filename="document.pdf", strategy="hi_res", # Use layout detection model extract_images_in_pdf=True, # Extract embedded images infer_table_structure=True, # Detect and structure tables include_page_breaks=True, ) text_elements = [e for e in elements if e.category in ("NarrativeText", "Title", "ListItem")] table_elements = [e for e in elements if e.category == "Table"] image_elements = [e for e in elements if e.category == "Image"]

Step 2: VLM Description for Indexing

def describe_image_for_rag(image_path: str, context: str = "") -> str: """Use VLM to create a text description of an image for embedding.""" with open(image_path, "rb") as f: img_b64 = base64.b64encode(f.read()).decode() response = client.chat.completions.create( model="gpt-4o-mini", # Cheaper model for descriptions messages=[{"role": "user", "content": [ {"type": "text", "text": f"""Describe this image in detail for a search index. Include: all text visible, data values, relationships shown, key takeaways. Document context: {context} Be specific and factual — this description will be used for retrieval."""}, {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{img_b64}"}}, ]}], max_tokens=500) return response.choices[0].message.content

Step 3: Build Multimodal Index

from langchain_chroma import Chroma from langchain_openai import OpenAIEmbeddings from langchain.schema import Document embeddings = OpenAIEmbeddings(model="text-embedding-3-small") documents = [] for elem in text_elements: documents.append(Document(page_content=elem.text, metadata={"type": "text", "page": elem.metadata.page_number})) for elem in table_elements: description = process_table_element(elem.metadata.text_as_html) documents.append(Document(page_content=description, metadata={"type": "table", "page": elem.metadata.page_number, "original_html": elem.metadata.text_as_html})) for elem in image_elements: description = describe_image_for_rag(elem.metadata.image_path) documents.append(Document(page_content=description, metadata={"type": "image", "page": elem.metadata.page_number, "image_path": elem.metadata.image_path})) vectorstore = Chroma.from_documents(documents, embeddings)

Step 4: Multimodal Retrieval + Generation

def multimodal_rag_query(question: str, vectorstore, k: int = 5) -> str: docs = vectorstore.similarity_search(question, k=k) text_context, images_to_send = [], [] for doc in docs: if doc.metadata["type"] == "text": text_context.append(doc.page_content) elif doc.metadata["type"] == "table": text_context.append(f"[Table]:\n{doc.metadata['original_html']}") elif doc.metadata["type"] == "image": text_context.append(f"[Image]: {doc.page_content}") images_to_send.append(doc.metadata["image_path"]) content = [{"type": "text", "text": f"Context:\n{'---'.join(text_context)}\n\nQ: {question}"}] for img_path in images_to_send[:3]: # Cost control: max 3 images with open(img_path, "rb") as f: content.append({"type": "image_url", "image_url": {"url": f"data:image/png;base64,{base64.b64encode(f.read()).decode()}"}}) response = client.chat.completions.create(model="gpt-4o", messages=[{"role": "system", "content": "Answer from context. Cite sources."}, {"role": "user", "content": content}], max_tokens=1000) return response.choices[0].message.content

Architecture Decision: Use "VLM description + text RAG + late fusion for top results" (hybrid). Pure text extraction loses too much from charts/diagrams. Pure multi-modal embeddings aren't reliable enough yet. The hybrid gives good retrieval (via text descriptions) PLUS high-quality answers (via sending actual images to VLM at generation time). Trade-off: higher indexing cost ($0.01-0.05/image) + query cost when images involved.
