Module 4, Topic 10 — Production Agents

Production agents need guardrails notebooks don't — step budgets, token caps, timeouts, fallback to simpler systems. The gap between "works in notebook" and "handles 1000 users at 3am" is where most agent projects fail. Default to chains (fixed pipelines); use agents only when tool selection or multi-step reasoning is genuinely needed. Every production agent must have: streaming, cost cap, timeout, and a fallback path.

Problem Statement: A notebook agent that works 90% of the time fails spectacularly in production — infinite loops burn credits, unhandled errors crash servers, and slow responses frustrate users. Production deployment requires engineering discipline: guardrails, streaming, fallback systems, and observability at every layer.

FastAPI Streaming Endpoint

from fastapi import FastAPI from fastapi.responses import StreamingResponse import json, time app = FastAPI(title="Agent API") @app.post("/chat/stream") async def chat_stream(query: str, thread_id: str = "default", max_steps: int = 10): config = {"configurable": {"thread_id": thread_id}} async def event_generator(): start = time.time() for event in agent.stream( {"messages": [HumanMessage(query)], "step_count": 0, "max_steps": max_steps}, config): if time.time() - start > 30: # Timeout guard yield f"data: {json.dumps({'type': 'error', 'content': 'Timed out'})}\n\n" break for node_name, output in event.items(): for msg in output.get("messages", []): if hasattr(msg, "tool_calls") and msg.tool_calls: yield f"data: {json.dumps({'type': 'tool_call', 'tool': msg.tool_calls[0]['name']})}\n\n" elif msg.content: yield f"data: {json.dumps({'type': 'response', 'content': msg.content})}\n\n" yield f"data: {json.dumps({'type': 'done', 'ms': int((time.time()-start)*1000)})}\n\n" return StreamingResponse(event_generator(), media_type="text/event-stream")

Human-in-the-Loop (HITL) Pattern

hitl_agent = graph.compile(checkpointer=checkpointer, interrupt_before=["tools"]) @app.post("/chat/hitl") async def chat_with_approval(query: str, thread_id: str): config = {"configurable": {"thread_id": thread_id}} result = hitl_agent.invoke( {"messages": [HumanMessage(query)], "step_count": 0, "max_steps": 10}, config) state = hitl_agent.get_state(config) if state.next: # Paused before tool execution pending = result["messages"][-1].tool_calls[0] return {"status": "awaiting_approval", "pending": {"tool": pending["name"], "args": pending["args"]}} return {"status": "complete", "response": result["messages"][-1].content} @app.post("/chat/approve") async def approve_action(thread_id: str, approved: bool): config = {"configurable": {"thread_id": thread_id}} if approved: result = hitl_agent.invoke(None, config) # Resume else: # Inject rejection message hitl_agent.update_state(config, {"messages": [ ToolMessage(content="Rejected by user.", tool_call_id=...)]}) result = hitl_agent.invoke(None, config) return {"response": result["messages"][-1].content}

Graceful Degradation (Fallback to Chain)

async def invoke_with_fallback(query: str, thread_id: str, max_steps: int = 10): """Agent first. If fails or hits budget, fall back to simple chain.""" config = {"configurable": {"thread_id": thread_id}} try: result = agent.invoke( {"messages": [HumanMessage(query)], "step_count": 0, "max_steps": max_steps}, config) if result.get("step_count", 0) >= max_steps: return await _fallback(query, "step_limit") return {"response": result["messages"][-1].content, "mode": "agent"} except Exception as e: return await _fallback(query, f"error:{type(e).__name__}") async def _fallback(query: str, reason: str): response = fallback_llm.invoke([ SystemMessage(content="Answer concisely. No tools available."), HumanMessage(content=query)]) return {"response": response.content, "mode": "fallback", "reason": reason}

Complexity Router (Chain vs Agent)

def select_execution_mode(query: str, user_tier: str) -> str: """Route to cheapest viable mode.""" if user_tier == "free": return "chain" needs_tools = any(kw in query.lower() for kw in ["search", "calculate", "look up", "find"]) needs_planning = len(query.split()) > 30 or "and then" in query.lower() if needs_planning: return "complex_agent" elif needs_tools: return "simple_agent" return "chain"

When NOT to Use an Agent

Use a CHAIN when:Use an AGENT when:
Task always follows same stepsTask varies by input
No tool selection neededMust choose between tools
Latency < 2s requiredUser tolerates 5-15s
Output format is fixedOutput depends on findings
70-80% of production queries20-30% that need reasoning
Architecture Decision: Default to simple chains. Use agents only when task requires tool selection or multi-step reasoning. MANDATORY guardrails on all agent deployments: step budget, timeout, cost cap, and fallback path. Agents are 5-50× more expensive, 3-10× slower, and less predictable. A complexity router saves agent overhead on 70-80% of queries. The gap between notebook and production is where most projects fail — solve it with guardrails, not hope.
