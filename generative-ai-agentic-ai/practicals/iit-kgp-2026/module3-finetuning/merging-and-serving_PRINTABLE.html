Module 3, Topic 9 — Merging & Serving

You have a trained adapter that passes evaluation — now deploy it. The gap between "model works in a notebook" and "model serves production traffic" is substantial. This topic covers merging LoRA into base weights, quantizing for efficient inference, choosing an inference engine (vLLM), and selecting a deployment target (serverless vs dedicated). Three key decisions: merge or not, quantize or not, where to deploy.

Problem: Your LoRA adapter is 10 MB but needs a 16 GB base model. You need to serve it efficiently — fast enough for users, cheap enough for budget, and reliable enough for production.

Deployment Pipeline

Deployment pipeline: adapter → merge → quantize → serve. Each step after merge is optional depending on your quality/cost trade-off.

Three Key Decisions

Implementation — Merge LoRA into Base

from peft import PeftModel from transformers import AutoModelForCausalLM, AutoTokenizer base_model = AutoModelForCausalLM.from_pretrained( "meta-llama/Llama-3-8B-Instruct", torch_dtype=torch.bfloat16, device_map="cpu") model = PeftModel.from_pretrained(base_model, "./lora-adapter") merged_model = model.merge_and_unload() # Merge adapter weights into base merged_model.save_pretrained("./merged-model") tokenizer.save_pretrained("./merged-model") # Result: complete model, no PEFT dependency needed for inference

Implementation — Quantize for Deployment

# Option A: AWQ (recommended — best quality at 4-bit) from awq import AutoAWQForCausalLM model = AutoAWQForCausalLM.from_pretrained("./merged-model") model.quantize(tokenizer, quant_config={"zero_point": True, "q_group_size": 128, "w_bit": 4}) model.save_quantized("./merged-model-4bit-awq") # Option B: GPTQ (alternative, slightly worse quality) from auto_gptq import AutoGPTQForCausalLM, BaseQuantizeConfig config = BaseQuantizeConfig(bits=4, group_size=128, desc_act=False) model = AutoGPTQForCausalLM.from_pretrained("./merged-model", quantize_config=config) model.quantize(calibration_dataset) model.save_quantized("./merged-model-4bit-gptq") # Option C: bitsandbytes (on-the-fly, simplest — no separate save) model = AutoModelForCausalLM.from_pretrained("./merged-model", quantization_config=BitsAndBytesConfig(load_in_4bit=True, bnb_4bit_quant_type="nf4"))
