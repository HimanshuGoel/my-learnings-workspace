Module 3, Topic 4 — Dataset Preparation

Dataset quality is the single biggest determinant of fine-tuning success. Your model will confidently replicate every error in training data. "Garbage in, garbage out" applies more here than anywhere else in ML — the model learns by adjusting weights to predict YOUR outputs, so every training example is a signal about what correct behavior looks like. High-quality examples = clear signal; noisy examples = averaged noise. This topic covers collection strategies, quality filtering, format requirements, and validation.

Problem: Fine-tuning is only as good as your data. One contradictory example teaches contradictory behavior. Too few examples cause overfitting. Too many mediocre examples produce mediocre models. You need a systematic approach to building datasets that produce reliable fine-tuned models.

Data Format Standards

OpenAI Chat Format (JSONL)

{"messages": [ {"role": "system", "content": "You are a customer support classifier."}, {"role": "user", "content": "My payment was charged twice last week."}, {"role": "assistant", "content": "Category: billing\nSeverity: high\nSummary: Double charge on payment\nAction: escalate to billing team"} ]}

HuggingFace Instruction Format

{"instruction": "Classify this customer ticket.", "input": "My payment was charged twice last week.", "output": "Category: billing\nSeverity: high\nSummary: Double charge on payment"}

Quality Filtering Pattern

def filter_dataset(examples: list[dict]) -> list[dict]: filtered = [] seen_inputs = set() for ex in examples: assistant_msg = ex["messages"][-1]["content"] user_msg = ex["messages"][-2]["content"] if len(assistant_msg) < 20: continue # Too short (incomplete) if len(assistant_msg) > 2000: continue # Too long (verbose/off-topic) if not has_expected_structure(assistant_msg): continue # Wrong format if hash(user_msg) in seen_inputs: continue # Duplicate input if contradicts_existing(ex, filtered): continue # Conflicting signal seen_inputs.add(hash(user_msg)) filtered.append(ex) return filtered

Dataset Validation Script

def validate_dataset(path: str) -> dict: examples = [json.loads(line) for line in open(path)] issues = [] if len(examples) < 50: issues.append(f"Too few: {len(examples)} (need 50+)") for i, ex in enumerate(examples): if "messages" not in ex: issues.append(f"Ex {i}: missing 'messages'") if len(ex.get("messages", [])) < 2: issues.append(f"Ex {i}: need user+assistant") lengths = [count_tokens(ex) for ex in examples] if max(lengths) > 4096: issues.append(f"Max tokens too high: {max(lengths)}") inputs = [ex["messages"][-2]["content"] for ex in examples] unique_ratio = len(set(inputs)) / len(inputs) if unique_ratio < 0.9: issues.append(f"Low diversity: {unique_ratio:.0%}") return {"total": len(examples), "avg_tokens": sum(lengths)//len(lengths), "unique_ratio": unique_ratio, "issues": issues, "valid": len(issues)==0}

Decision Checkpoint

☐ Dataset format matches target model (messages array or instruction format)
☐ Every example is correct — I'd be happy if the model produced this exact output
☐ Dataset is diverse enough (90%+ unique inputs, covers all categories)
☐ No contradictions between examples (same input → same output)
☐ Proper train/val/test split with no leakage
☐ Validation script passes with zero issues
☐ Dataset size is appropriate for task complexity
☐ I have a plan to iterate if first training run shows data issues
