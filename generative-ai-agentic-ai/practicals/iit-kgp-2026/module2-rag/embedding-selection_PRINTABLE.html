Module 2, Topic 3 — Embedding Selection

Embedding models convert text into dense vectors where semantic similarity maps to geometric closeness — this is how retrieval works. You understood this conceptually in Module 1. Now the engineering question: which model, what dimensions, what trade-offs, and how do you debug when retrieval quality doesn't meet expectations?

Problem: Dozens of embedding models with different dimensions, speed, cost, and quality. Wrong choice = poor retrieval (cheap model) or wasted spend (overkill model). You can't change models later without re-indexing your entire corpus.

Decision Flowchart

Pick the simplest model that meets your needs. Upgrade only with measured evidence of quality gap.

Implementation — Key Code Patterns

Development Setup (sentence-transformers)

from sentence_transformers import SentenceTransformer model = SentenceTransformer("all-MiniLM-L6-v2") doc_embeddings = model.encode( documents, batch_size=64, normalize_embeddings=True, # cosine sim = dot product when normalized ) query_embedding = model.encode("search query", normalize_embeddings=True)

Production with OpenAI API

from openai import OpenAI client = OpenAI() response = client.embeddings.create( input=texts, model="text-embedding-3-small", # dimensions=512, # optional: reduce dims (saves storage, small quality loss) ) embeddings = [item.embedding for item in response.data]

BGE Models — Query Prefix Required

# BGE/E5 models need instruction prefix for QUERIES (not documents) queries = ["Represent this sentence for searching relevant passages: " + q for q in raw_queries] query_embs = model.encode(queries, normalize_embeddings=True) # Documents embedded WITHOUT prefix doc_embs = model.encode(documents, normalize_embeddings=True) # ⚠ Missing the prefix = significant retrieval quality drop!

LangChain Integration

# Local from langchain_community.embeddings import HuggingFaceEmbeddings embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2") # OpenAI from langchain_openai import OpenAIEmbeddings embeddings = OpenAIEmbeddings(model="text-embedding-3-small") # Use with vector store vectorstore = Chroma.from_documents(documents, embeddings)

Critical Architecture Constraint

Same model, same version for both indexing and querying. Model change = full re-index (expensive).

Failure Modes → Fixes

Retrieval quality suddenly dropsQuery embedded with different model than docsVerify same model for index + query
Symmetric queries work, asymmetric don'tMissing query prefix (BGE, E5 models)Add "Represent this sentence..." prefix
Long documents get bad embeddingsText exceeds model's max seq length (silently truncated)Chunk documents properly before embedding
API cost unexpectedly highRe-embedding unchanged documentsCache embeddings, only re-embed changed content

Multilingual content retrieves poorlyModel trained on English onlyUse multilingual: Cohere v3, multilingual-MiniLM
Near-duplicate resultsEmbeddings too similar (low dims)Try higher-dim model, or add metadata filters
