Module 2, Topic 8 — Production RAG

Moving from notebook to production means handling everything notebooks ignore: concurrent users, network failures, cost control, cold starts, graceful degradation, and observability. Production RAG is a FastAPI service with streaming responses, semantic caching for cost reduction, cascading fallbacks so users never see a blank screen, and Prometheus metrics so you know when things degrade. This is the "deploy it" topic — your bridge to Module 5 (Deployment).

Problem: Pipeline works in notebook but fails in production — cold starts, cost spiraling, silent failures, no observability. Production needs caching, streaming, fallbacks, monitoring, and cost controls.

Notebook vs Production Gap

Every notebook pattern has a production equivalent. Bridge the gap systematically, not all at once.

Production Architecture

Standard topology: LB → stateless service replicas → external stores + APIs + monitoring.

FastAPI Service with Streaming

@app.post("/query/stream") async def query_stream(request: QueryRequest): async def generate(): docs = await retriever.ainvoke(request.question) sources = [doc.metadata["source"] for doc in docs] yield f"data: {json.dumps({'type': 'sources', 'data': sources})}\n\n" messages = build_prompt(request.question, docs) async for chunk in llm.astream(messages): if chunk.content: yield f"data: {json.dumps({'type': 'token', 'data': chunk.content})}\n\n" yield f"data: {json.dumps({'type': 'done'})}\n\n" return StreamingResponse(generate(), media_type="text/event-stream")

Semantic Caching

class SemanticCache: """Cache by query SIMILARITY, not exact match. 30-60% cost reduction.""" def __init__(self, vectorstore, threshold=0.95, ttl_hours=24): self.vectorstore = vectorstore self.threshold = threshold # 0.95 = very similar queries async def get(self, query: str) -> dict | None: results = self.vectorstore.similarity_search_with_score(query, k=1) if results and results[0][1] >= self.threshold: return self.cache_store[results[0][0].metadata["cache_key"]] return None async def set(self, query: str, response: dict): cache_key = hashlib.md5(query.encode()).hexdigest() self.vectorstore.add_texts([query], metadatas=[{"cache_key": cache_key}]) self.cache_store[cache_key] = {"response": response, "timestamp": now()}

Cascading Fallback Chain

async def resilient_query(query: str): try: # Level 1: Full pipeline return await asyncio.wait_for(full_pipeline(query), timeout=10.0) except (RetrievalError, asyncio.TimeoutError): pass try: # Level 2: Skip reranking return await no_rerank_pipeline(query) except Exception: pass try: # Level 3: Direct LLM (no retrieval) return await direct_llm(query) except Exception: pass return {"answer": "I'm having issues. Please try again shortly.", # Level 4 "sources": [], "fallback_level": "apologize"} # NEVER: unhandled exception, blank screen, hard crash

Rate Limiting & Cost Control

class CostTracker: PRICING = { "gpt-4o-mini": {"input": 0.15/1e6, "output": 0.60/1e6}, "text-embedding-3-small": {"input": 0.02/1e6}, } def record(self, model, input_tokens, output_tokens=0): cost = input_tokens * self.PRICING[model]["input"] + \ output_tokens * self.PRICING[model].get("output", 0) self.daily_cost += cost if self.daily_cost > self.daily_budget: raise BudgetExceeded(f"${self.daily_budget} exceeded!")

Health & Readiness Probes

@app.get("/health") # Liveness: is process alive? async def health(): return {"status": "ok"} @app.get("/ready") # Readiness: can I handle queries? async def ready(): checks = {} try: vectorstore.similarity_search("test", k=1); checks["vectordb"] = "ok" except: checks["vectordb"] = "error" try: await llm.ainvoke("test"); checks["llm"] = "ok" except: checks["llm"] = "error" ok = all(v == "ok" for v in checks.values()) return JSONResponse(status_code=200 if ok else 503, content={"ready": ok, ...})
