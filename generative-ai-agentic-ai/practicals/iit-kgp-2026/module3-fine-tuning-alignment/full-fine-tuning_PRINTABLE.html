Module 3, Topic 2 — Full Fine-Tuning (SFT)

Supervised Fine-Tuning (SFT) is the foundation all other techniques build on. You show the model (input, correct_output) pairs and adjust ALL weights to make it produce those outputs. Think of it as the "full recompile" of model behavior — expensive but thorough. In practice, you'll usually use LoRA instead (Topic 3), but understanding full SFT gives you the mental model for everything that follows.

Problem: You need the model to reliably produce your specific output format/style. SFT is how you go from "general model" to "your task specialist."

How SFT Works

SFT training loop: forward pass → loss → backprop → weight update. All 8B parameters are modified.

Key insight: LoRA achieves 90-98% of full SFT quality at 10% of the cost. Full SFT is the reference implementation — understand it, but use LoRA in practice.

Memory Budget Formula

Full SFT (bf16): 2B/param (weights) + 2B/param (grads) + 8B/param (optimizer) = 12B/param → 8B model = 8 × 12 = 96 GB minimum (need multi-GPU) LoRA (bf16 base): 2B/param (frozen weights) + tiny adapter = ~18 GB for 8B model → Fits on single A100 40GB QLoRA (4-bit): 0.5B/param (quantized) + adapter = ~8 GB for 8B model → Fits on T4 16GB!

Implementation — HuggingFace (Full Control)

from transformers import AutoModelForCausalLM, TrainingArguments, Trainer model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3-8B-Instruct", torch_dtype=torch.bfloat16) args = TrainingArguments( output_dir="./output", num_train_epochs=3, per_device_train_batch_size=4, gradient_accumulation_steps=4, learning_rate=2e-5, warmup_ratio=0.1, weight_decay=0.01, bf16=True, gradient_checkpointing=True, # saves ~60% memory eval_strategy="steps", eval_steps=50, save_steps=100, ) trainer = Trainer(model=model, args=args, train_dataset=..., eval_dataset=...) trainer.train()

What to Watch — Loss Curves

Always plot both train and eval loss. The divergence point tells you when to stop training.

Failure Modes → Fixes

ProblemSymptomFix
OverfittingTrain loss ↓, eval loss ↑ after epoch 2Reduce epochs, more data, add dropout
Catastrophic forgettingGeneral tasks (math, code) degradeMix 10-20% general instruction data
LR too highLoss spikes, NaN, garbage outputReduce LR by 2-5×, add warmup
LR too lowLoss barely moves over epochsIncrease LR by 2-5×
