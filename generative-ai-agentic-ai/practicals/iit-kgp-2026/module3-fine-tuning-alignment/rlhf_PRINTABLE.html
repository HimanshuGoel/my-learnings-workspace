Module 3, Topic 6 — RLHF

RLHF (Reinforcement Learning from Human Feedback) is how models go from "competent text predictor" to "helpful assistant." It encodes human preferences into a reward signal, then optimizes the model to maximize that reward while staying close to SFT behavior. This is what made ChatGPT feel helpful rather than just capable. You need to understand it conceptually, but most practitioners will use DPO (simpler, cheaper) instead of implementing RLHF directly.

Problem: SFT teaches imitation (copy this output), but doesn't teach WHY one response is better than another. RLHF teaches the model to optimize for human preference — helpfulness, safety, quality — going beyond mere imitation to genuine alignment.

3-Stage Pipeline Architecture

RLHF 3-stage pipeline: SFT teaches format → Reward model encodes preferences → PPO optimizes for preference scores.

Why RLHF > SFT Alone

AspectSFT OnlySFT + RLHF
Learns fromCorrect examples onlyPreferences (this > that)
Quality ceilingAverage of your examplesCan exceed your examples
SafetyCan produce unsafe textTrained to avoid unsafe outputs
HelpfulnessFollows instructionsActively maximizes helpfulness
Failure modeCopies style, may miss intentOptimizes for satisfaction

The KL Penalty — Critical Constraint

# PPO optimization without KL penalty: # Model finds "reward hacks" — outputs that score high but aren't actually good # Example: repeating "I'm happy to help!" gets high reward score → model spams it # KL penalty constrains HOW MUCH the model can change from SFT: Loss = -reward + β × KL(policy || reference_model) # β = 0.1-0.3: Higher = more conservative (less change from SFT) # If model drifts too far → KL penalty dominates → pulls it back # Result: model improves within bounds of reasonable behavior

Reward Hacking — The Main Failure Mode

Hack TypeWhat HappensWhy It Scores HighPrevention
Length gamingGenerates very long responsesReward model biased toward lengthLength-normalize rewards
RepetitionRepeats high-scoring phrasesExploits reward model weaknessIncrease KL penalty
SycophancyAlways agrees with userAgreement scored as "helpful"Train reward model on disagreement examples
Format gamingUses bullet points/bold excessivelyStructured text scores higherDiverse reward training data
