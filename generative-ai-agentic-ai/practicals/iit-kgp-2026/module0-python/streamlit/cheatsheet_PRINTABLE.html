Streamlit — Complete Learning Notes

Summary: Streamlit builds interactive web UIs for ML apps in pure Python — one file, no frontend code. Write a script → get a web app. The instant demo UI for your capstone.

Why Streamlit Exists

  • Problem — your AI app is a Python script or notebook. Stakeholders can't run Python. You need a demo UI but don't want to learn React.
  • Solution — write Python top-to-bottom. Streamlit converts it to an interactive web page. Every widget = one line. User interacts → script reruns.
  • Mental Model — one-file React app for data. Script reruns on every interaction (like React re-render). session_state = React state. @cache = useMemo. No components, no hooks, no JSX.
  • AI Connection — the standard demo frontend for ML projects. 30 lines → ChatGPT-like interface over your docs.

1. Core Execution Model

Every interaction re-runs the entire script. session_state persists between runs. @cache avoids recomputation.
Key mental model: Script runs top-to-bottom on every click/input. Use session_state to persist data. Use @cache_resource to avoid reloading models.

2. Essential Patterns

Display

st.title("Title") st.write("anything") st.dataframe(df) st.metric("Acc", "92%", "+3%") st.code("x = 1", language="python") st.pyplot(fig) # matplotlib

Input

name = st.text_input("Name") text = st.text_area("Message") n = st.slider("K", 1, 10, 3) choice = st.selectbox("Pick", opts) on = st.checkbox("Enable") file = st.file_uploader("Upload") if st.button("Go"): do_thing()

Layout

col1, col2 = st.columns(2) with st.sidebar: st.selectbox(...) tab1, tab2 = st.tabs(["A","B"]) with st.expander("More"): ...

State

if "key" not in st.session_state: st.session_state.key = default st.session_state.key = new_value

Cache

@st.cache_data # data (df, str) @st.cache_resource # models, DB conn

Status

with st.spinner("Loading..."): ... st.success("Done!") st.error("Failed!")

3. Chat Interface (The Capstone Pattern)

if "messages" not in st.session_state: st.session_state.messages = [] for msg in st.session_state.messages: with st.chat_message(msg["role"]): st.write(msg["content"]) if prompt := st.chat_input("Ask"): st.session_state.messages.append({"role": "user", "content": prompt}) with st.chat_message("user"): st.write(prompt) with st.chat_message("assistant"): with st.spinner("Thinking..."): response = rag_chain.invoke(prompt) st.write(response) st.session_state.messages.append({"role": "assistant", "content": response})
This is your capstone template. Replace rag_chain.invoke(prompt) with your actual LangChain/LlamaIndex call. Everything else stays the same.

5. Common Mistakes

#MistakeFix
1Variables reset on every interactionUse st.session_state for persistent data
2Model reloads on every rerun (slow)Use @st.cache_resource for models
3No loading indicator for slow opsWrap in with st.spinner("..."):
4One giant script (unmaintainable)Use pages/ folder for multi-page apps
5Not using chat_message for chat appsst.chat_message + st.chat_input = proper chat UI
