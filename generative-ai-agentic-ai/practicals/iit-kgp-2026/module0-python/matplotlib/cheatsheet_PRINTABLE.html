Matplotlib — Complete Learning Notes

Summary: Matplotlib turns numbers into charts — it's Python's foundational visualization library. Every plot you see in ML papers, Kaggle notebooks, and presentations is Matplotlib (or something built on top of it).

Why Matplotlib Exists

  • Problem — raw numbers (arrays, tables) are hard to interpret. A 1000-row dataset tells you nothing by staring at it. You need visual patterns.
  • Solution — Matplotlib renders arrays as charts (line, bar, scatter, heatmap) so humans can spot distributions, outliers, trends, and model behavior at a glance.
  • Mental Model — the HTML Canvas API for data. Figure = browser window. Axes = individual canvas element. You draw on each Axes with method calls. Style with parameters instead of CSS.
  • AI Connection — training blind is debugging without logs. Matplotlib is how you see overfitting, class imbalance, feature distributions, and model errors.

Where Matplotlib Fits

Matplotlib is for human understanding — not computation. It sits between data and decisions.

1. Figure & Axes — The Object Hierarchy

Everything in Matplotlib is a Figure containing Axes. Understanding this hierarchy is the key to avoiding pyplot spaghetti.

Figure = container. Axes = where you draw. One figure can have many axes (subplots).
# ALWAYS use the OO interface (not plt.plot shortcut) fig, ax = plt.subplots(figsize=(10, 6)) # single plot fig, axes = plt.subplots(2, 3, figsize=(14, 8)) # 2×3 grid fig, (ax1, ax2) = plt.subplots(1, 2) # 1×2 unpacked
Rule: Never use plt.plot() for anything beyond a quick one-off. Always use fig, ax = plt.subplots() and draw on ax. This scales to complex layouts.

3. Customization — Making Plots Professional

A chart without labels is useless. Every plot needs: title, axis labels, legend (if multiple series), and appropriate sizing.

# Essential customization (do this EVERY time) ax.set_title("Training Progress", fontsize=12, fontweight="bold") ax.set_xlabel("Epoch") ax.set_ylabel("Loss") ax.legend(loc="upper right") ax.grid(True, alpha=0.3) # Save high-quality output fig.savefig("plot.png", dpi=150, bbox_inches="tight") plt.tight_layout() # fix overlapping labels between subplots

4. Common ML Plot Recipes

These 5 visualizations appear in almost every ML project. Memorize the patterns.

Training Loss Curve

fig, ax = plt.subplots(figsize=(8, 5)) ax.plot(epochs, train_loss, label="Train", color="#4F46E5") ax.plot(epochs, val_loss, label="Validation", color="#f59e0b") ax.set_xlabel("Epoch"); ax.set_ylabel("Loss") ax.set_title("Training Progress") ax.legend(); ax.grid(True, alpha=0.3)
  • What to look for: If val_loss rises while train_loss falls → overfitting. If both stay high → underfitting.

Confusion Matrix

im = ax.imshow(cm, cmap="Blues") for i in range(cm.shape[0]): for j in range(cm.shape[1]): ax.text(j, i, str(cm[i,j]), ha="center", va="center") ax.set_xticks(range(n)); ax.set_yticks(range(n)) ax.set_xticklabels(classes); ax.set_yticklabels(classes) ax.set_xlabel("Predicted"); ax.set_ylabel("Actual")

Feature Importance

sorted_idx = importances.argsort()[::-1][:10] ax.barh(features[sorted_idx][::-1], importances[sorted_idx][::-1]) ax.set_xlabel("Importance"); ax.set_title("Top Features")

Distribution Check

ax.hist(data, bins=30, edgecolor="black", alpha=0.7) ax.axvline(data.mean(), color="red", linestyle="--", label="Mean") ax.legend()

Class Balance

counts = df["target"].value_counts() ax.bar(counts.index.astype(str), counts.values) ax.set_title("Class Distribution")

5. Subplots & Layout

# Grid of subplots fig, axes = plt.subplots(2, 3, figsize=(14, 8)) axes[0, 0].plot(...) # access by [row, col] axes[1, 2].bar(...) # Flatten for easy iteration axes = axes.flatten() for i, col in enumerate(columns): axes[i].hist(df[col]) # Always call at the end plt.tight_layout() fig.savefig("output.png", dpi=150, bbox_inches="tight")

7. Common Mistakes

#MistakeFix
1Using plt.plot() for complex figures (spaghetti)Use OO: fig, ax = plt.subplots()
2Labels/titles cut off or overlappingCall plt.tight_layout() before save/show
3Default figsize too smallStart with figsize=(10, 6)
4Using pie charts for comparisonUse bar charts — humans compare lengths better than angles
5Overplotting (too many dots overlapping)Use alpha=0.3 or hexbin/density
6Memory leak in loops (not closing figures)plt.close(fig) after savefig
7No axis labels — chart is uninterpretableAlways add xlabel, ylabel, title

Note: Seaborn is NOT a replacement — it's a layer built ON TOP of Matplotlib. Learn the OO interface first; Seaborn then becomes a one-line shortcut for statistical plots.

Mental Models Summary

  • Figure — browser window / page
  • Axes — individual canvas element to draw on
  • plot() — draw a line (trends, time series)
  • scatter() — draw dots (relationships)
  • bar() — draw rectangles (comparisons)
  • hist() — draw distribution shape
  • imshow() — draw a colored grid (matrices)
  • subplots() — create a grid of drawing areas
  • tight_layout() — auto-fix spacing (call every time)
  • savefig() — export to file (PNG for web, PDF for print)
  • OO interface — work with DOM elements directly (more control)
  • Seaborn — jQuery on top of Matplotlib (shortcuts, defaults)
