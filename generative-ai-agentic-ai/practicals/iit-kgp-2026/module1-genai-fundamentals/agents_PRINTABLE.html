Topic 11 — Agents

Summary: An AI agent is an LLM in a reasoning loop with access to tools — it autonomously decides which actions to take (search, execute code, query DBs) to accomplish multi-step goals.

Why It Exists

  • Problem — standard LLMs can only generate text. They can't check real-time info, execute actions, verify their answers, or break complex tasks into executable steps.
  • Solution — give the LLM access to tools (functions it can call) and a reasoning loop (think → act → observe → repeat). The model decides WHICH tool and WHAT arguments.
  • Mental Model — A junior developer with access to Stack Overflow, a terminal, and docs. They think about what to do, search for info, try an approach, check if it worked, iterate until done.
  • Or: Like a Kubernetes controller (reconciliation loop). Observe current state → decide action needed → execute → observe new state → loop until desired state achieved.

How It Works

The ReAct pattern: Think → Act (tool call) → Observe (result) → Repeat until goal is achieved.

ReAct agent loop: Think → Act (call tool) → Observe (result) → loop until goal achieved, then respond

Tool Definition Pattern

@tool def search_web(query: str) -> str: """Search the internet for current information.""" ← LLM reads this description return search_api.query(query) @tool def query_database(sql: str) -> str: """Run a SQL query against the customer database.""" return db.execute(sql) # LLM sees: tool name + description + parameter schema → decides which to call and with what args
Key insight: The tool description IS the documentation for the LLM. Write them like you'd write JSDoc for a junior dev. Clear, concise, with examples of when to use it. 3-7 well-defined tools is optimal — too many confuses the model.

Common Misconceptions

WrongCorrect
"Agents are autonomous AI"They're LLMs in a loop with tools. Still need guardrails, limits, and human oversight.
"Agents always succeed"They can get stuck in loops, call wrong tools, or hallucinate tool arguments.
"More tools = better agent"Too many tools confuse the model. 3-7 well-defined tools is optimal.
"Agents replace human judgment"They handle routine multi-step tasks. Complex decisions still need humans.
"Agents need fine-tuning"Most agents work with prompted base models + good tool definitions. No fine-tuning needed.

Top Takeaways

#Insight
1Agent = LLM + Tools + Loop. The "intelligence" is the LLM deciding what to do next.
2ReAct pattern (Think → Act → Observe → Repeat) is the most common agent architecture.
3Tools are just functions with good descriptions. LLM reads the description to decide when to call.
43-7 well-defined tools is optimal. Too many = decision paralysis for the model.
5Agents can fail: infinite loops, wrong tool calls, hallucinated arguments. Always add guardrails.
6Standard LLM = single pass. Agent = multiple reasoning steps. More powerful but less predictable.
7RAG, calculators, code execution — these are all just tools an agent can use.
