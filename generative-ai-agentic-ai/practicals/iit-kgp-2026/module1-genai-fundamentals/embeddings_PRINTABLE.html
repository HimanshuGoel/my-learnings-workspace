Topic 3 — Embeddings

Summary: Embeddings are dense vectors representing meaning in geometric space — similar concepts live close together, different concepts live far apart. The foundation of semantic search and RAG.

Why It Exists

  • Problem — after tokenization, you have IDs like [348, 14372]. But 348 and 349 aren't semantically related. Need a representation where relationships are geometric.
  • One-hot fails — 50K-dimensional sparse vector. "Cat" and "kitten" equally far from each other as "cat" and "telescope." No similarity.
  • Solution — learn a 768-float vector per token where similar meanings = nearby vectors. Distance = semantic difference.
  • Mental Model — GPS coordinates for meaning. Nearby GPS = nearby places. Nearby embeddings = nearby meanings.

How It Works

Embedding = learned lookup table. Token ID → row in matrix → dense vector representing meaning.

Similarity in Embedding Space

In embedding space, semantically related words form clusters. "cat" is near "kitten" but far from "python" (programming).

Static vs Contextual Embeddings

TypeExample"bank" in "river bank""bank" in "bank account"
Static (Word2Vec)One fixed vectorSame vector ✗Same vector ✗
Contextual (BERT)Depends on sentenceNature-related vector ✓Finance-related vector ✓
Modern standard: Always contextual (BERT, GPT). Static embeddings (Word2Vec) are outdated. Context determines meaning.

The RAG Connection

RAG = embed docs + embed query + find similar using cosine similarity. No embeddings = no RAG.

Common Misconceptions

WrongCorrect
Each dimension has a clear meaningMeaning is distributed across all 768 dims — no single "happiness" axis
Embeddings are fixedContextual (modern): same word gets different vectors in different sentences
Bigger = better384-dim often sufficient. 768 adds cost with diminishing returns for most tasks.
Embeddings capture truthThey capture statistical patterns — including biases in training data
