Topic 8 — Decoding Strategies

Summary: Decoding is how a language model selects which token to output next from its predicted probability distribution — controlling the trade-off between creativity and precision.

Why It Exists

  • Problem — after a forward pass, the model outputs probabilities over 50,000+ tokens. How do you choose ONE token from that distribution?
  • Solution — decoding strategies (temperature, top-p, top-k) let you control randomness: deterministic for code, creative for brainstorming.
  • Mental Model — Volume knob for creativity. Like randomness in game procedural generation: seed + randomness params control whether you get "safe expected" or "surprising creative" output.
  • Or: Like log-level configuration. You dial between TRACE (verbose/random) and ERROR (focused/deterministic) depending on what you need.

How It Works

Model produces logits → apply strategy → select token → repeat autoregressively.

Decoding pipeline: model outputs logits → softmax → decoding strategy → selected token

Temperature Effect on Probability Distribution

Temperature controls distribution shape: lower = peaked/deterministic, higher = flat/creative

Decoding Strategies Comparison

Top-P (Nucleus Sampling) Explained

Probabilities: [0.40, 0.25, 0.15, 0.08, 0.05, 0.03, 0.02, ...] Top-P = 0.9: 0.40 + 0.25 + 0.15 + 0.08 = 0.88 (not enough) 0.40 + 0.25 + 0.15 + 0.08 + 0.05 = 0.93 ≥ 0.9 → keep top 5 tokens Sample from these 5 (renormalized) Why adaptive? If the model is confident (one token at 0.9), top-p keeps only 1. If uncertain (flat dist), it keeps many. Adapts to context automatically.

Practical Settings Guide

Use CaseTemperatureTop-PRationale
Code generation0.0–0.21.0Precision matters, creativity is bugs
Factual QA0.0–0.30.9Consistent, accurate answers
Creative writing0.7–1.00.9Diversity and surprise wanted
Brainstorming1.0–1.50.95Maximum exploration
Classification0.01.0Deterministic — same input = same output
Key insight: Temperature and top-p are independent knobs. Temperature shapes the distribution, top-p truncates it. Most APIs let you set both. Start with temperature for coarse control, add top-p for safety net.

Common Misconceptions

WrongCorrect
"Temperature=0 means no randomness"Technically makes distribution infinitely peaked → equivalent to greedy. Same effect, different mechanism.
"Higher temperature = smarter"Higher temperature = more random. Can be better OR worse depending on task.
"Top-P and Top-K are the same"Top-K = fixed count. Top-P = adaptive (includes more tokens when model is uncertain).
"Just use default settings"Defaults (T=1, no top-p) are rarely optimal. Always tune for your task.
"Beam search is always better"Beam search is slower and can produce bland text. Sampling often sounds more natural.

Key Terminology

TermMeaning
LogitsRaw model output scores (before softmax) for each token in vocabulary
SoftmaxConverts logits to probabilities that sum to 1
TemperatureDivides logits before softmax — controls distribution sharpness
Top-KKeep only K highest-probability tokens, zero out the rest
Top-P (Nucleus)Keep tokens until cumulative probability reaches P threshold
Greedy decodingAlways pick the single highest-probability token (T≈0)
Beam searchTrack N best partial sequences simultaneously, pick best overall
SamplingRandomly pick from the (possibly filtered) probability distribution

Top Takeaways

#Insight
1Decoding doesn't change the model — it changes how we sample from its output distribution
2Temperature is the primary knob: low (0–0.3) for precision, high (0.7+) for creativity
3Top-P is adaptive (adjusts candidate count to model confidence), Top-K is fixed
4Every API call (OpenAI, Anthropic, etc.) exposes temperature + top_p as parameters
5For code/factual tasks: T=0. For creative tasks: T=0.7–1.0. Always tune per use case.
6Beam search is great for translation/summarization but produces bland conversational text
7Prompt determines WHAT logits look like; decoding determines HOW to select from them
