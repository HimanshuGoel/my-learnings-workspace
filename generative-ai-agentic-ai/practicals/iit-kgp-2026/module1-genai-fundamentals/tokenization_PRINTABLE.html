Topic 2 — Tokenization

Summary: Tokenization converts raw text into integer IDs using subword splitting — the bridge between human language and what a neural network can process.

Why It Exists

  • Problem — neural networks only process numbers, not strings. Need a consistent text → integers mapping.
  • Word-level fails — vocabulary explodes (millions of words). Can't handle typos, new words, other languages.
  • Character-level fails — sequences become extremely long (1000-char sentence = 1000 tokens). Model loses word meaning.
  • Solution — subword tokenization: split into frequent chunks. "unhappiness" → "un" + "happiness". Balances vocab size with sequence length.
  • Mental Model — UTF-8 for AI. Variable-length encoding: common text → short representation, rare text → split into smaller pieces.

How BPE (Byte Pair Encoding) Works

BPE: train by merging frequent pairs → use by applying learned merges → output integer IDs

Tokenization Levels Compared

Subword tokenization finds the sweet spot between vocabulary size and sequence length

Key Properties

  • 1 word ≈ 1.3 tokens (on average for English). "Extraordinary" = 3 tokens, "the" = 1 token.
  • Spaces are part of tokens — " is" (with space) is different from "is" (without).
  • Context window = max tokens — GPT-4 has 128K token context. Your prompt + output must fit.
  • Cost = per token — API billing counts input + output tokens. Shorter prompts = cheaper.

Critical rule: Always use the tokenizer that matches your model. GPT tokenizer + BERT model = garbage. They're paired — the vocabulary must match.
