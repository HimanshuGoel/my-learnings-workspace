Module 4, Topic 6 — Multi-Agent Systems

Multi-agent coordinates specialized LLM agents for complex problems. BUT 90% of apps don't need this — a single agent with tools handles most cases. Multi-agent is justified only when single-agent quality is demonstrably insufficient AND tasks naturally decompose into separate expertise domains. Each additional agent multiplies latency, cost, and debugging complexity.

Problem Statement: Some tasks require different types of expertise (research, code, writing) that are hard to combine in one system prompt, or need quality validation where one agent checks another's work. Multi-agent provides this — but only use it when you've proven a single agent can't handle the task class.

Supervisor Implementation (LangGraph)

from langgraph.graph import StateGraph, END from langchain_openai import ChatOpenAI from typing import TypedDict, Annotated from operator import add class MultiAgentState(TypedDict): messages: Annotated[list, add] next_agent: str def research_agent(state: MultiAgentState) -> dict: llm = ChatOpenAI(model="gpt-4o-mini") response = llm.invoke([ {"role": "system", "content": "You are a research specialist."}, *state["messages"]]) return {"messages": [response]} def code_agent(state: MultiAgentState) -> dict: llm = ChatOpenAI(model="gpt-4o-mini") response = llm.invoke([ {"role": "system", "content": "You are a coding specialist."}, *state["messages"]]) return {"messages": [response]} def supervisor(state: MultiAgentState) -> dict: """Route to the right specialist or finish.""" llm = ChatOpenAI(model="gpt-4o-mini") decision = llm.invoke([ {"role": "system", "content": "Route: research | code | FINISH"}, *state["messages"]]) # Parse decision from response return {"next_agent": parse_routing(decision.content)} def route_supervisor(state: MultiAgentState) -> str: return state["next_agent"] graph = StateGraph(MultiAgentState) graph.add_node("supervisor", supervisor) graph.add_node("research", research_agent) graph.add_node("code", code_agent) graph.set_entry_point("supervisor") graph.add_conditional_edges("supervisor", route_supervisor, { "research": "research", "code": "code", "FINISH": END}) graph.add_edge("research", "supervisor") # After specialist → back to supervisor graph.add_edge("code", "supervisor") multi_agent = graph.compile()

Architecture Decision: Default to single agent + tools. Multi-agent only when: (1) Single agent quality is demonstrably insufficient (measured, not assumed), (2) Task naturally decomposes into independent expertise domains, (3) Latency of 10-30s is acceptable, (4) You have engineering capacity to maintain and debug. Multi-agent adds 2-5× latency, N× cost, and exponential debugging complexity.
