Module 2, Topic 1 — Advanced Prompting

Prompting is how you program LLMs — your system prompt, few-shot examples, and instructions ARE the "code" that determines output behavior. Module 1 covered the concept (zero-shot, few-shot, CoT). Here you're engineering production prompts: systematic design, version control, evaluation, and the judgment of which technique to use for which situation in a RAG pipeline.

Problem: Basic prompts produce inconsistent, unstructured outputs. You need prompts that handle edge cases, produce deterministic formats, and degrade gracefully.

Chain-of-Thought: When to Use vs Skip

CoT adds latency and cost — use only when the task genuinely requires multi-step reasoning

Production Considerations

ConcernApproachImpact
CostPrompt caching (system + examples cached)~50% cost reduction on repeat calls
LatencyAvoid CoT for simple queries; use streamingTTFT: 200ms (simple) vs 800ms (CoT)
VersioningYAML files, git-tracked, A/B testedRollback in seconds if new prompt degrades
MonitoringLog prompt hash + format compliance rateCatch regressions within 1 hour
