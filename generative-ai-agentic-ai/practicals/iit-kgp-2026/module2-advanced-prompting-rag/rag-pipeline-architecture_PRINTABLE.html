Module 2, Topic 6 — RAG Pipeline Architecture

A RAG pipeline chains together query processing, retrieval, re-ranking, prompt assembly, and generation into one end-to-end system. Like any software architecture, the goal is composability — each stage independently testable, replaceable, and observable. This is where the previous topics (chunking, embeddings, vector stores, retrieval) come together into a shippable system.

Problem: You have individual components (chunker, embedder, vector store, retriever, LLM) but no coherent system. Without intentional architecture, you get untestable, unmonitorable spaghetti. You need composable stages with clear boundaries.

End-to-End Pipeline Architecture

5-stage pipeline. Each stage independently testable, measurable, and replaceable. Streaming hides generation latency.

Implementation — Stage by Stage

Stage 1: Query Processing

class ProcessedQuery(BaseModel): original: str intent: Literal["factual", "howto", "comparison", "troubleshoot"] expanded_queries: list[str] # 3 variants for multi-query retrieval target_collection: str metadata_filters: dict def process_query(query: str, llm) -> ProcessedQuery: intent = classify_intent(query, llm) # "What is..." → factual expanded = expand_query(query, llm, n=3) # Generate 3 rephrased variants collection = route_to_collection(intent) # factual → api_docs filters = extract_filters(query) # "v2.1" → {"version": "2.1.0"} return ProcessedQuery(...)

Stage 2: Retrieval (Hybrid + Multi-Query)

class RetrievalStage: def retrieve(self, processed_query): all_docs = [] for variant in processed_query.expanded_queries: docs = self.hybrid_retriever.invoke(variant) # BM25 + vector all_docs.extend(docs) # Deduplicate by content hash unique = deduplicate(all_docs) return unique[:20] # Cap at 20 for reranker

Stage 3: Reranking

class RerankingStage: def rerank(self, query: str, docs: list) -> list: pairs = [(query, doc.page_content) for doc in docs] scores = self.cross_encoder.predict(pairs) scored = sorted(zip(scores, docs), reverse=True) # Filter by min score AND take top_n return [doc for score, doc in scored if score >= 0.3][:3]

Stage 4: Augmentation (Prompt Assembly)

SYSTEM = """You are a technical documentation assistant. Rules: 1) Answer ONLY from context 2) If not found, say so 3) Cite [Source: filename]""" def build_prompt(query, docs): context = "\n\n".join([ f"[{i}] (Source: {d.metadata['source']})\n{d.page_content}" for i, d in enumerate(docs, 1) ]) return [("system", SYSTEM), ("human", f"Context:\n{context}\n\nQuestion: {query}")]

Stage 5: Generation (Streaming)

class GenerationStage: def __init__(self): self.llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0) async def generate_stream(self, messages): async for chunk in self.llm.astream(messages): yield chunk.content # Token-by-token to client

Full Orchestration

class RAGPipeline: def run(self, user_query: str) -> RAGResponse: processed = self.query_processor.process(user_query) # Stage 1 candidates = self.retrieval.retrieve(processed) # Stage 2 top_docs = self.reranking.rerank(processed.original, candidates) # Stage 3 messages = self.augmentation.build_prompt(processed.original, top_docs) # Stage 4 return self.generation.generate(messages) # Stage 5

Conversational RAG (Adding Memory)

from langchain.memory import ConversationBufferWindowMemory memory = ConversationBufferWindowMemory( memory_key="chat_history", return_messages=True, k=5 # last 5 turns ) # Memory is injected between system prompt and user message # Enables follow-up questions: "What about its auth?" → knows "it" = previous topic
