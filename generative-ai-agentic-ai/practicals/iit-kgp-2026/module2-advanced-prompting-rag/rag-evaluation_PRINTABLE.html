Module 2, Topic 7 — RAG Evaluation

RAG evaluation measures three things: did you retrieve the right documents (context quality), is the answer grounded in those documents (faithfulness/no hallucination), and does the answer actually address the question (relevancy). RAGAS is the industry-standard framework that automates these checks using LLM-as-judge. Without evaluation, you're flying blind — every change to chunking, retrieval, or prompts could silently degrade quality.

Problem: "It seems to work" isn't engineering. You need measurable metrics for retrieval quality, answer faithfulness, and relevancy — with automated testing that catches regressions before users do.

The RAG Evaluation Triangle

Three relationships RAGAS evaluates: Did we retrieve right docs? Is answer grounded in context? Does it address the question?

RAGAS Core Metrics

Two most important: Faithfulness catches hallucination. Context Recall catches retrieval failures. If you measure only two things, measure these.

Implementation — RAGAS Setup

Prepare Evaluation Dataset

from ragas import evaluate from ragas.metrics import faithfulness, answer_relevancy, context_precision, context_recall from datasets import Dataset eval_data = { "question": ["What is the rate limit?", "How to auth with OAuth?"], "answer": ["100 req/min [Source: api.md]", "OAuth 2.0 client credentials [Source: auth.md]"], "contexts": [["API rate limit is 100 requests/min per key."], ["Auth uses OAuth 2.0. Flows: client credentials, auth code."]], "ground_truth": ["100 requests per minute per API key.", "OAuth 2.0 with client creds or auth code."], } dataset = Dataset.from_dict(eval_data)

Run Evaluation

results = evaluate( dataset=dataset, metrics=[faithfulness, answer_relevancy, context_precision, context_recall], ) print(results) # {'faithfulness': 0.92, 'answer_relevancy': 0.88, 'context_precision': 0.85, 'context_recall': 0.90}

Custom LLM-as-Judge

JUDGE_PROMPT = """Score the RAG answer from 1-5: 5 = Perfect: accurate, complete, cited 4 = Good: minor omission 3 = Acceptable: correct but incomplete 2 = Poor: partially wrong 1 = Bad: hallucinated or off-topic Question: {question} Context: {context} Answer: {answer} Reference: {ground_truth} Return JSON: {"score": <1-5>, "reasoning": "", "issues": ["..."]}""" judge = ChatOpenAI(model="gpt-4o", temperature=0) # Use strongest model for judging result = judge.invoke(JUDGE_PROMPT.format(...))

Regression Testing Pattern

class RAGRegressionTest: def run(self, pipeline, test_set, thresholds): dataset = self._run_pipeline_on_test_set(pipeline, test_set) results = evaluate(dataset=dataset, metrics=[faithfulness, answer_relevancy, ...]) # Check against thresholds for metric, threshold in thresholds.items(): if results[metric] < threshold: raise RegressionError(f"{metric}: {results[metric]:.3f} < {threshold}") self._save_to_history(results) # Track trending over time return results

A/B Testing Pattern

class ABTest: def route(self, user_id: str) -> str: # Deterministic: same user always gets same variant hash_val = int(hashlib.md5(user_id.encode()).hexdigest(), 16) return "a" if (hash_val % 100) < 50 else "b" def run(self, query, user_id): variant = self.route(user_id) pipeline = self.pipeline_a if variant == "a" else self.pipeline_b result = pipeline.run(query) self.log(query, variant, result) # For later analysis return result # Run for 1-2 weeks or 500+ queries for statistical significance
