Module 2, Topic 5 — Retrieval Quality

In RAG, the LLM can only work with what you feed it. Retrieval is the bottleneck — if the right documents don't make it into the context window, no amount of prompt engineering or model quality can save the answer. This topic covers the engineering techniques to systematically improve what comes back: re-ranking, hybrid search, query expansion, and how to diagnose which technique you actually need.

Problem: RAG retrieves documents but answers are wrong — right docs not in top-K (recall) or too many irrelevant docs dilute context (precision). Retrieval quality is the #1 lever: bad retrieval → bad answers, regardless of LLM quality.

Diagnosis Decision Tree

Diagnose systematically. Each technique addresses a specific failure mode — don't add all at once.

Before vs After Optimization

Over-retrieve then filter gives better results than trying to get top-K perfect in one step.

Implementation — Key Patterns

Baseline: Vector + Metadata Filter

retriever = vectorstore.as_retriever( search_type="similarity", search_kwargs={"k": 5, "filter": {"doc_type": "api_doc"}} )

MMR (Quick Win for Diversity)

retriever = vectorstore.as_retriever( search_type="mmr", search_kwargs={ "k": 5, "fetch_k": 20, "lambda_mult": 0.7, # 0=max diversity, 1=max relevance (0.5-0.7 sweet spot) } )

Re-Ranking with Cross-Encoder

from langchain.retrievers import ContextualCompressionRetriever from langchain.retrievers.document_compressors import CrossEncoderReranker from langchain_community.cross_encoders import HuggingFaceCrossEncoder cross_encoder = HuggingFaceCrossEncoder(model_name="cross-encoder/ms-marco-MiniLM-L-6-v2") reranker = CrossEncoderReranker(model=cross_encoder, top_n=3) # Retrieve 20 → rerank → return top 3 base_retriever = vectorstore.as_retriever(search_kwargs={"k": 20}) reranking_retriever = ContextualCompressionRetriever( base_compressor=reranker, base_retriever=base_retriever )
Why cross-encoder > bi-encoder for reranking: Bi-encoder embeds Q and D separately (fast but less accurate). Cross-encoder processes Q+D as a PAIR (slow but much more accurate). Use bi-encoder for initial retrieval, cross-encoder for reranking.

Hybrid Search (BM25 + Vector)

from langchain.retrievers import EnsembleRetriever from langchain_community.retrievers import BM25Retriever bm25 = BM25Retriever.from_documents(documents, k=10) vector = vectorstore.as_retriever(search_kwargs={"k": 10}) hybrid = EnsembleRetriever( retrievers=[bm25, vector], weights=[0.3, 0.7], # semantic dominates, BM25 catches keywords )

Multi-Query Expansion

from langchain.retrievers.multi_query import MultiQueryRetriever multi_query = MultiQueryRetriever.from_llm( retriever=vectorstore.as_retriever(search_kwargs={"k": 5}), llm=ChatOpenAI(model="gpt-4o-mini", temperature=0.3), ) # "OAuth auth" → also searches "How to implement OAuth 2.0", "authentication setup"

Full Pipeline: Hybrid + Rerank

# Best practice: combine hybrid (recall) + reranking (precision) hybrid = EnsembleRetriever(retrievers=[bm25, vector], weights=[0.3, 0.7]) final = ContextualCompressionRetriever( base_compressor=reranker, base_retriever=hybrid ) # Keyword coverage + semantic understanding + precision reranking
