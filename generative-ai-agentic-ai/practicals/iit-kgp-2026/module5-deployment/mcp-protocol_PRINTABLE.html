Module 5, Topic 2 — MCP Protocol

Model Context Protocol (MCP) is an emerging standard for AI tool integration — letting AI assistants (Claude, GPT, Kiro, Cursor) connect to external tools and data sources via a standardized server interface. Instead of building tool integrations per assistant, you build ONE MCP server that works everywhere. It's how you make your Module 4 tools interoperable across any MCP-compatible client.

Problem: Every AI IDE/assistant has its own tool-calling mechanism. MCP provides a universal protocol (JSON-RPC 2.0 over stdio/HTTP) so your tools become discoverable and invocable by any compliant AI assistant — without rewrites per client.

MCP Architecture

MCP architecture: Host (AI IDE) → Client (protocol layer) → Server (your tools/resources/prompts) → Backend services.

Tool Registration Flow

Tool registration: Client reads config → starts server → discovers tools → invokes on demand → returns results to LLM context.

MCP Server Implementation

# mcp_server/server.py from mcp.server import Server from mcp.server.stdio import stdio_server from mcp.types import Tool, TextContent, Resource import json server = Server("my-ai-tools") @server.list_tools() async def list_tools() -> list[Tool]: """Declare available tools for AI assistants to discover.""" return [ Tool( name="search_documents", description="Search internal knowledge base using semantic search. Returns relevant chunks with source.", inputSchema={ "type": "object", "properties": { "query": {"type": "string", "description": "Natural language search query"}, "top_k": {"type": "integer", "description": "Results to return", "default": 5}, "filter_source": {"type": "string", "enum": ["confluence", "github", "docs"]} }, "required": ["query"] } ), Tool( name="query_database", description="Execute read-only SQL queries against analytics DB. Only SELECT allowed.", inputSchema={ "type": "object", "properties": {"sql": {"type": "string", "description": "SQL SELECT query"}}, "required": ["sql"] } ), ] @server.call_tool() async def call_tool(name: str, arguments: dict) -> list[TextContent]: """Execute tool and return results.""" match name: case "search_documents": results = await search_vectors(arguments["query"], arguments.get("top_k", 5)) return [TextContent(type="text", text=json.dumps(results, indent=2))] case "query_database": if not arguments["sql"].strip().upper().startswith("SELECT"): return [TextContent(type="text", text="Error: Only SELECT queries allowed.")] rows = await execute_readonly_query(arguments["sql"]) return [TextContent(type="text", text=json.dumps(rows, indent=2))]

Resource Exposure

# Expose data sources the AI can read for additional context @server.list_resources() async def list_resources() -> list[Resource]: return [ Resource( uri="docs://architecture/overview", name="System Architecture Overview", description="High-level architecture document", mimeType="text/markdown" ), ] @server.read_resource() async def read_resource(uri: str) -> str: resource_map = {"docs://architecture/overview": "docs/architecture.md"} file_path = resource_map.get(str(uri)) if not file_path: raise ValueError(f"Resource not found: {uri}") with open(file_path) as f: return f.read()

HTTP Transport (Production)

# mcp_server/http_transport.py from mcp.server.sse import SseServerTransport from starlette.applications import Starlette from starlette.routing import Route, Mount sse = SseServerTransport("/messages/") async def handle_sse(request): async with sse.connect_sse(request.scope, request.receive, request._send) as streams: await server.run(streams[0], streams[1], server.create_initialization_options()) routes = [Route("/sse", endpoint=handle_sse), Mount("/messages/", app=sse.handle_post_message)] app = Starlette(routes=routes) # Run: uvicorn mcp_server.http_transport:app --host 0.0.0.0 --port 8080

Security Best Practices

  • Read-only DB user: Connection should ONLY have SELECT permissions
  • Input validation: Validate all tool arguments against schema before execution
  • Secret management: Never log API tokens; use environment variables with ${VAR} syntax
  • Audit logging: Log every tool call — who invoked, what arguments, what returned
  • HTTP auth: Remote transport MUST require Bearer token + HTTPS
✅ MCP Deployment Checklist:
□ Server starts cleanly: python -m mcp_server with no errors
□ mcp dev inspector shows all tools with correct schemas
□ Each tool has clear description (AI uses this to decide WHEN to call)
□ Input schemas have proper types, descriptions, and required fields
□ DB tools use read-only connections (never write via MCP)
□ Secrets via env vars (not hardcoded in server code)
□ Error handling in call_tool (never crash the server)
□ HTTP transport uses authentication for remote access
□ Tool responses are concise (don't dump 10MB to the AI context)
□ Tested with actual AI client (Claude Desktop / Kiro), not just inspector
