Module 5, Topic 5 — Authentication & Security

AI services face unique security threats beyond traditional web APIs: prompt injection (malicious instructions in user input), jailbreaking (bypassing safety guardrails), data exfiltration (extracting training data or system prompts), and cost attacks (forcing expensive operations). Your standard auth/rate-limiting knowledge applies — but you also need AI-specific defenses.

What you're deploying: API key authentication, per-user rate limiting, input sanitization, prompt injection defense, and output filtering for your AI service.

Security Layers

Defense in depth: 5 layers. Even if one layer is bypassed, others protect the system.

Implementation — Rate Limiting

from collections import defaultdict import time class RateLimiter: def __init__(self, max_requests: int = 60, window_seconds: int = 60): self.max_requests = max_requests self.window = window_seconds self.requests = defaultdict(list) # In production: use Redis def check(self, user_id: str) -> bool: now = time.time() # Remove old entries self.requests[user_id] = [t for t in self.requests[user_id] if now - t < self.window] if len(self.requests[user_id]) >= self.max_requests: return False self.requests[user_id].append(now) return True

from pydantic import BaseModel, Field, validator class SecureChatRequest(BaseModel): message: str = Field(min_length=1, max_length=10000) @validator("message") def sanitize_message(cls, v): # Strip null bytes, control chars v = v.replace("\x00", "").strip() # Check length in tokens (not just chars) if count_tokens(v) > 2000: raise ValueError("Message too long (max 2000 tokens)") # Check for injection patterns if detect_injection(v): raise ValueError("Input contains disallowed patterns") return v

Pre-Deploy Checklist

☐ API key authentication on all endpoints (no anonymous access)
☐ Rate limiting active (requests/min + tokens/day per user)
☐ Input validation: max length, sanitization, injection detection
☐ Output filtering: moderation API + PII redaction
☐ HTTPS enforced, CORS restricted to known origins
☐ System prompt not extractable (tested with extraction prompts)
☐ Secrets in env vars / secrets manager (never in code/logs)
☐ Audit log: who called what, when (NOT full message content)
☐ Key rotation policy defined (90-day expiry)
