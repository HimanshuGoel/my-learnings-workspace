Module 5, Topic 10 — Capstone Guide

The capstone is your chance to demonstrate everything: RAG, agents, deployment, evaluation, responsible AI — integrated into ONE working system. Scope is the hardest part. Most failures come from over-scoping (too ambitious), not under-scoping (too simple). A polished, well-documented system that does ONE thing excellently scores higher than a sprawling prototype that does five things poorly.

What you're building: A complete, deployed AI system that demonstrates mastery across Modules 1-5. Feedback session Jan 30, final evaluation Feb 6.

Capstone Timeline

5-week timeline. Start building Week 1, not Week 4. The most common mistake is delaying implementation.

Scope — What to Build

AmbitionExample ProjectModules DemonstratedDifficulty
Safe bet (recommended)Domain-specific RAG chatbot with streaming + eval2, 4, 5Medium
AmbitiousRAG + agent with tools + fine-tuned model + deployment2, 3, 4, 5High

Scope rule: Pick ONE core capability and do it EXCELLENTLY. Then add layers (eval, monitoring, security) for depth. A polished RAG chatbot with full eval + monitoring + responsible AI scores higher than a half-working multi-agent system.

Common Mistakes to Avoid

MistakeWhy It HappensFix
Over-scopingWant to show ALL modulesDo ONE thing excellently with depth, not breadth
No evaluation"It works in demo" feels sufficient50+ test cases with RAGAS is non-negotiable
Starting lateModule 5 topics first, capstone laterStart building Week 1, apply topics AS you learn them
No error handlingWorks in happy path onlyTest: empty query, long query, adversarial query, tool failure
No documentationCode speaks for itself (it doesn't)README with setup, architecture diagram, eval results
Demo fails liveDepends on API that's downRecord backup video, have cached responses ready
