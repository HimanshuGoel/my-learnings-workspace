Module 5, Topic 4 — Streaming & UX

LLM generation takes 2-10 seconds. Without streaming, users stare at a blank screen. With streaming (Server-Sent Events), the first token appears in <500ms and text flows in real-time — making 5s feel like 1s. For agents, stream intermediate steps ("Searching...", "Analyzing...") so users know the system is working. This is the single biggest UX improvement for AI apps.

What you're deploying: Real-time token streaming via SSE, intermediate agent step notifications, loading states, and error recovery UX for your FastAPI AI service.

Streaming Architecture

SSE flow: client sends POST, server yields events as LLM generates. Client renders tokens incrementally.

Implementation — Server Side

from fastapi import APIRouter from fastapi.responses import StreamingResponse import json @router.post("/chat/stream") async def chat_stream(request: ChatRequest): async def event_generator(): try: # Emit sources first (retrieved docs) docs = await retriever.ainvoke(request.message) yield f"data: {json.dumps({'type':'sources','data':[d.metadata['source'] for d in docs]})}\n\n" # Stream LLM tokens messages = build_prompt(request.message, docs) async for chunk in llm.astream(messages): if chunk.content: yield f"data: {json.dumps({'type':'token','data':chunk.content})}\n\n" yield f"data: {json.dumps({'type':'done'})}\n\n" except Exception as e: yield f"data: {json.dumps({'type':'error','data':str(e)})}\n\n" return StreamingResponse(event_generator(), media_type="text/event-stream")

Implementation — Client Side (JavaScript)

const response = await fetch('/v1/chat/stream', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({message: query}) }); const reader = response.body.getReader(); const decoder = new TextDecoder(); let buffer = ''; while (true) { const {done, value} = await reader.read(); if (done) break; buffer += decoder.decode(value, {stream: true}); const lines = buffer.split('\n\n'); buffer = lines.pop(); // Keep incomplete for (const line of lines) { if (!line.startsWith('data: ')) continue; const event = JSON.parse(line.slice(6)); if (event.type === 'token') appendToUI(event.data); else if (event.type === 'tool_call') showThinking(`Using ${event.data}...`); else if (event.type === 'done') hideLoading(); else if (event.type === 'error') showError(event.data); } }

Agent Streaming — Intermediate Steps

# For LangGraph agents: stream node executions async def agent_stream(query: str): async for event in agent.astream_events( {"messages": [HumanMessage(query)]}, config, version="v2"): if event["event"] == "on_chat_model_stream": yield sse_event("token", event["data"]["chunk"].content) elif event["event"] == "on_tool_start": yield sse_event("thinking", f"Using {event['name']}...") elif event["event"] == "on_tool_end": yield sse_event("tool_result", event["data"]["output"][:100])

Pre-Deploy Checklist

☐ SSE endpoint returns text/event-stream content type
☐ Nginx/LB configured with proxy_buffering off
☐ Keep-alive comments sent every 15s for long responses
☐ Error events sent (not just dropped connection) on failure
☐ Client handles reconnection gracefully (no duplicate content)
☐ Timeout configured (60s max stream duration)
☐ Agent intermediate steps emitted as events (thinking, tool_call)
☐ "Done" event signals completion (client stops waiting)
☐ Mobile/flaky network tested (disconnect + reconnect scenario)
