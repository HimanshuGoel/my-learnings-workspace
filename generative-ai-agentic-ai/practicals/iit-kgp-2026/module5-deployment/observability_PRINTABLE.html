Module 5, Topic 9 — Observability

Observability answers "what is my AI system doing right now, and why?" Traditional monitoring (is it up? how fast?) isn't enough for AI — you also need to trace through agent decision chains, see which tools were called, measure quality drift, and debug why a specific response was bad. LangSmith provides AI-native tracing; Prometheus/Grafana handles infrastructure metrics.

What you're deploying: Three pillars of observability — metrics (Prometheus), logs (structured), and traces (LangSmith) — tailored for AI system debugging.

Observability Stack

Three pillars work together: metrics tell you SOMETHING is wrong, logs tell you WHAT, traces tell you WHY.

LangSmith Setup (AI-Native Tracing)

# 1. Set environment variables: import os os.environ["LANGCHAIN_TRACING_V2"] = "true" os.environ["LANGCHAIN_API_KEY"] = "ls__..." # from smith.langchain.com os.environ["LANGCHAIN_PROJECT"] = "my-ai-service-prod" # 2. That's it. All LangChain/LangGraph calls are now auto-traced. # Each trace shows: # - Full message history at each step # - Token counts (input/output) per LLM call # - Tool inputs/outputs # - Latency per node # - Error details with stack trace # 3. Add custom metadata per request: config = { "metadata": {"user_id": "user-42", "endpoint": "/chat"}, "tags": ["production", "tier-free"], } result = agent.invoke(input, config)

Prometheus Metrics

from prometheus_client import Counter, Histogram, Gauge, start_http_server # Define metrics request_count = Counter("ai_requests_total", "Total requests", ["endpoint", "status"]) request_latency = Histogram("ai_request_duration_seconds", "Latency", ["endpoint"], buckets=[0.5, 1, 2, 5, 10, 30]) tokens_used = Counter("ai_tokens_total", "Tokens consumed", ["model", "direction"]) active_requests = Gauge("ai_active_requests", "Currently processing") cost_total = Counter("ai_cost_usd_total", "Cumulative cost", ["model"]) # In middleware: @app.middleware("http") async def metrics_middleware(request, call_next): active_requests.inc() start = time.time() response = await call_next(request) duration = time.time() - start request_latency.labels(endpoint=request.url.path).observe(duration) request_count.labels(endpoint=request.url.path, status=response.status_code).inc() active_requests.dec() return response # Expose metrics endpoint: start_http_server(9090) # Prometheus scrapes this

Structured Logging

import structlog logger = structlog.get_logger() # Per-request log (NOT message content — privacy): logger.info("request_completed", request_id="req-abc123", user_id="user-42", endpoint="/v1/chat", model="gpt-4o-mini", tokens_in=450, tokens_out=120, latency_ms=1830, tools_called=["search_docs"], cache_hit=False, safety_flagged=False, ) # Error log: logger.error("llm_call_failed", request_id="req-abc123", error_type="RateLimitError", retry_count=2, fallback_used=True, )

Debugging with Traces

# When a user reports "bad answer": # 1. Find the trace in LangSmith by request_id or timestamp # 2. See exactly what was retrieved (retrieval node output) # 3. See what prompt was assembled (augmentation node) # 4. See what the LLM generated (generation node) # 5. Identify: Was it bad retrieval? Bad prompt? Bad generation? # Debugging decision tree: # Retrieved docs irrelevant? → Retrieval problem (chunking, embedding, query) # Docs relevant but answer wrong? → Prompt problem (instructions, context format) # Prompt good but answer hallucinated? → Model problem (temp, decoding, model choice) # Module 1 connection: tokenization/decoding knowledge helps diagnose step 5

Troubleshooting

SymptomCauseFix
LangSmith traces missingEnv var not set or API key wrongVerify LANGCHAIN_TRACING_V2=true and API key valid
Prometheus scrape failingMetrics port not exposedExpose 9090 in Docker, verify /metrics endpoint
Logs too verboseLogging at DEBUG in productionSet LOG_LEVEL=info, only DEBUG for specific investigations
Can't correlate trace to user complaintNo request_id in response headersReturn X-Request-ID header, user includes in bug report
Trace storage costs too highTracing 100% of requestsSample: trace 10% in production, 100% on errors

Pre-Deploy Checklist

☐ LangSmith tracing configured (LANGCHAIN_TRACING_V2=true)
☐ Prometheus metrics: latency, error rate, token usage, cost
☐ Structured logging: request metadata (not content) per request
☐ Grafana dashboards: service health + AI quality + agent behavior
☐ Alerts configured: error rate, latency, cost, step limit
☐ Request ID in response headers (for user-reported issues)
☐ Sampling configured (10% traces in prod, 100% on error)
☐ Retention policy: 30d detailed traces, 90d metrics
☐ Can debug a "bad answer" report using trace within 5 minutes
