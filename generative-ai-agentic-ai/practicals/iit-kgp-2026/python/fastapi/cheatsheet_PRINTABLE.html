FastAPI — Complete Learning Notes

Summary: FastAPI is Spring Boot for AI Engineers — build production REST APIs with auto-validation, auto-docs, and async support. The standard way to deploy ML models and RAG systems as HTTP services.

Why FastAPI Exists

  • Problem — your trained model/RAG pipeline is a Python function. It needs to become an HTTP service that frontends, mobile apps, and other services can call.
  • Solution — decorators for routes, Pydantic for validation, auto-generated OpenAPI docs, async support for concurrent requests — all in ~20 lines of code.
  • Mental Model — Spring Boot for AI Engineers. Route decorators = @RestController. Pydantic models = @Valid DTOs. Auto-docs = Swagger built-in. Dependency injection = @Autowired.
  • AI Connection — the deployment layer for Module 5 capstone. Wrap your RAG/agent/model as HTTP endpoints.

Where FastAPI Fits

FastAPI wraps your AI logic as HTTP endpoints — any client can call it over the network

1. Core Pattern

from fastapi import FastAPI from pydantic import BaseModel, Field app = FastAPI() class QueryRequest(BaseModel): question: str = Field(min_length=1) top_k: int = Field(default=3, ge=1, le=10) class QueryResponse(BaseModel): answer: str confidence: float @app.post("/ask", response_model=QueryResponse) def ask(request: QueryRequest): # auto-validated! return QueryResponse(answer="...", confidence=0.9) # Run: uvicorn main:app --reload # Docs: http://localhost:8000/docs (auto-generated!)
That's the entire pattern: Pydantic model → decorator → function → return model. FastAPI handles validation, serialization, error responses, and OpenAPI docs automatically.

2. Load Model at Startup (Lifespan)

from contextlib import asynccontextmanager @asynccontextmanager async def lifespan(app: FastAPI): app.state.model = load_model() # load ONCE at startup app.state.index = build_index() yield # app runs here # cleanup on shutdown app = FastAPI(lifespan=lifespan) @app.post("/predict") def predict(): model = app.state.model # already loaded!

3. Async vs Sync

TypeWhenExample
async defI/O-bound (API calls, DB, files)LLM API call, vector DB query
def (sync)CPU-bound (computation)sklearn predict, NumPy math
Rule: If you await something, use async def. If it's pure computation, use plain def. Mixing wrong = blocked event loop.

6. Common Mistakes

#MistakeFix
1Loading model inside endpoint (per-request!)Use lifespan to load once at startup
2Blocking async with sync codeUse plain def for CPU work
3Forgetting CORS (frontend blocked)Add CORSMiddleware
4No response_model (leaks internal data)Always declare response_model
5Hardcoded configUse Pydantic Settings + env vars
6No error handling (500 with stacktrace)HTTPException for expected, handler for unexpected

Top Takeaways

#Insight
1FastAPI = Pydantic models + route decorators + auto-docs (that's the core)
2Load models in lifespan, NOT in endpoints (load once, use forever)
3/docs gives you free Swagger UI — test endpoints without writing a frontend
4response_model prevents leaking internal data (passwords, raw DB fields)
5StreamingResponse enables ChatGPT-like token streaming
6Use async def for API/DB calls, plain def for CPU computation
7FastAPI is the standard for ML model serving in Python (replaced Flask)
8Your Module 5 capstone = FastAPI + RAG + Streamlit frontend
