PyTorch — Complete Learning Notes

Summary: PyTorch builds and trains neural networks with GPU acceleration and automatic gradient computation. It's the engine behind HuggingFace Transformers, Stable Diffusion, and most modern AI research.

Why PyTorch Exists

  • Problem — traditional ML (sklearn) can't learn hierarchical patterns from raw images/text. Deep learning can, but requires GPU compute and automatic differentiation across millions of parameters.
  • Solution — PyTorch provides tensors (GPU-enabled arrays) + autograd (automatic calculus) + nn.Module (composable building blocks) with a Pythonic, debug-friendly interface.
  • Mental Model — Angular + NumPy + Automatic Calculus. Tensors = NumPy arrays that track math history. Autograd = calculates gradients for you. nn.Module = Angular Components you compose. Or: if sklearn is Spring Boot (magic .fit()), PyTorch is Express.js — you write the loop, you have full control.
  • AI Connection — every modern LLM (GPT, BERT, LLaMA) is a PyTorch nn.Module. HuggingFace Transformers is a wrapper around PyTorch.

Where PyTorch Fits

PyTorch is the foundation — everything in modern AI is built on top of it

1. Tensors — NumPy Arrays with Superpowers

  • What — multi-dimensional arrays (like NumPy) that can run on GPU and track computation history for automatic differentiation.
  • Why not just NumPy? — NumPy can't: (a) run on GPU, (b) automatically compute gradients. Both are essential for training neural networks.
  • Conversion — torch.from_numpy(arr) / tensor.numpy() — shares memory (zero-copy).
t = torch.randn(3, 4) # random tensor t.shape # torch.Size([3, 4]) t.dtype # torch.float32 t.device # cpu (or cuda:0) # GPU: tensor.to("cuda") | CPU: tensor.to("cpu") # NumPy ↔ Torch: shared memory, instant conversion

2. Autograd — Automatic Differentiation

The reason PyTorch exists. It computes gradients (derivatives) automatically — essential for training neural networks.

Autograd records forward ops, then computes gradients in reverse — no manual calculus needed
x = torch.tensor([2.0], requires_grad=True) y = x**2 + 3*x # forward y.backward() # backward (auto-compute dy/dx) print(x.grad) # tensor([7.]) — the gradient

3. nn.Module — Building Neural Networks

Every neural network in PyTorch is an nn.Module. It's the Component class of the framework.

nn.Module = Angular Component. __init__ = template setup. forward() = render method.
class MyModel(nn.Module): def __init__(self): super().__init__() self.net = nn.Sequential( nn.Linear(784, 128), nn.ReLU(), nn.Dropout(0.3), nn.Linear(128, 10)) def forward(self, x): return self.net(x)

4. The Training Loop — Heart of Deep Learning

Unlike sklearn's .fit(), PyTorch makes the loop explicit. This gives full control over every step.

The 4 steps of every training iteration: forward → loss → backward → update weights
for epoch in range(num_epochs): for X_batch, y_batch in train_loader: optimizer.zero_grad() # clear old gradients output = model(X_batch) # forward pass loss = criterion(output, y_batch) # compute loss loss.backward() # compute gradients optimizer.step() # update weights
Don't forget: optimizer.zero_grad() before backward! Gradients accumulate by default — omitting this causes exploding updates.

6. Evaluation & Inference

model.eval() # disable dropout, batch norm with torch.no_grad(): # don't track gradients (saves memory) output = model(test_data) preds = output.argmax(dim=1) model.train() # re-enable for next training
Two things to remember for inference: (1) model.eval() — changes dropout/batchnorm behavior. (2) torch.no_grad() — saves memory and speeds up computation.

8. Common Mistakes

#MistakeFix
1Forgetting optimizer.zero_grad()Always zero before backward (gradients accumulate!)
2Not switching train/eval modemodel.train() for training, model.eval() for inference
3Forgetting torch.no_grad() at inferenceWrap eval code in with torch.no_grad():
4Device mismatch (model GPU, data CPU).to(device) both model AND data
5Applying softmax before CrossEntropyLossCrossEntropyLoss includes softmax — pass raw logits
6Learning rate too highStart with 1e-3 for Adam, reduce if loss oscillates
