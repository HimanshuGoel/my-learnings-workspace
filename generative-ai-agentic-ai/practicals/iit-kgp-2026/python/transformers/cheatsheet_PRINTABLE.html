HuggingFace Transformers — Complete Learning Notes

Summary: HuggingFace Transformers gives instant access to thousands of pre-trained AI models (BERT, GPT, LLaMA) with a unified API — load, infer, fine-tune in a few lines. It's npm for AI models.

Why HuggingFace Exists

  • Problem — training a state-of-the-art NLP model from scratch costs millions of dollars and months of compute. Most AI teams can't afford this.
  • Solution — a registry of pre-trained models (the Hub) + a unified API to load, use, and fine-tune them. Like npm install but for AI models.
  • Mental Model — npm/Maven for AI models. Hub = registry. from_pretrained() = install. pipeline() = quickstart. If PyTorch is Express.js, HuggingFace is Next.js — batteries included but ejectable to raw PyTorch.
  • AI Connection — this IS where modern AI lives. GPT, BERT, LLaMA, Whisper, CLIP — all accessible through this library.

Where HuggingFace Fits

HuggingFace is the source of models for everything downstream — RAG, agents, deployment

1. pipeline() — One-Line Inference

The fastest way to use any model. One line to load and infer. Use for prototyping and demos.

from transformers import pipeline pipeline("sentiment-analysis")("I love this!") # → POSITIVE 0.99 pipeline("zero-shot-classification")(text, labels) # → classify without training pipeline("text-generation", model="gpt2")(prompt) # → generate text pipeline("summarization")(long_text) # → short summary pipeline("question-answering")(question=q, context=ctx) # → extract answer pipeline("feature-extraction")(text) # → get embeddings
When to use pipeline(): Quick prototyping, demos, small-scale inference. For production or fine-tuning, use the explicit AutoModel + AutoTokenizer approach.

2. Tokenizer — Text to Numbers

Models don't understand text. The tokenizer converts "Hello" → [7592] (a token ID). Always use the tokenizer that matches your model.

Tokenizer splits text into subwords, maps to IDs, adds [CLS]/[SEP] — this is the model's input
tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased") encoded = tokenizer("Hello", return_tensors="pt", padding=True, truncation=True, max_length=128) # encoded["input_ids"], encoded["attention_mask"] → pass to model

Embeddings convert text to vectors for semantic search. This is the retrieval step in RAG.

7. LoRA / PEFT — Efficient Fine-Tuning (Module 3)

  • Problem — fine-tuning ALL parameters of a 7B model requires 28GB+ VRAM and hours of compute.
  • Solution — LoRA freezes the original model and trains tiny adapter matrices (0.1-1% of parameters). Nearly same quality, fraction of cost.
from peft import LoraConfig, get_peft_model config = LoraConfig(r=8, lora_alpha=16, target_modules=["query","value"]) model = get_peft_model(model, config) # Now only ~0.4% of parameters are trainable!

9. Common Mistakes

#MistakeFix
1Wrong tokenizer for model (different vocab = garbage)Always load tokenizer + model from same checkpoint
2Forgetting return_tensors="pt" (returns lists, not tensors)Always pass return_tensors="pt" for PyTorch
3Not padding/truncating batches (crash on different lengths)padding=True, truncation=True, max_length=N
4OOM on large modelsUse smaller model, reduce batch, quantize (4/8-bit)
5Full fine-tuning when LoRA is enoughPEFT trains 0.1% params with similar quality
6Confusing model types (CausalLM vs Classification)Match AutoModelForXxx to your task
