# Best Practices

- For technical writing, prefer the doc-as-code approach as it lives with the project source code itself.

- The content should have accuracy, clarity and conciseness (like can have TL;DR section). Use present tense sentences.

- Content should have headings and subheadings, bulleted and numbered lists for key points, bold and italics for important words, short paragraphs and clear visual hierarchy.

- Content should have visual elements like screenshots, icons, illustrations, charts, diagrams, logos, images, drawings, symbols, signs and maps.

- Good examples to refer for technical writing -
  - <https://github.com/hashicorp/terraform/tree/main/docs>
  - <https://github.com/fastapi/fastapi/tree/master/docs>
